# AD-Research 宏观研究模块体验评审报告

**评审视角**：10 年以上中美欧宏观研究经验的资深研究员  
**评审模块**：宏观 `/macro`、全球速览 `/global`、板块轮动 `/sector-rotation`、首页看板 `/dashboard`  
**评审日期**：2026-07-16  
**评审方式**：基于本地代码仓库（`/Users/aidanliu/Documents/Coding-Project/Investment-Research-Platform`）静态审查，并确认本地 dev server（frontend `:5173` / backend `:8000`）已启动；因 API 需认证，未做全量数据实证。

---

## 1. 总体评价

平台宏观数据基建已经比较扎实：

- 统一 `macro_indicator` 表 + `(code, region, period, source)` 唯一键，可干净聚合 FRED、akshare、yfinance 三源；
- `/global` 页面已覆盖主要跨境市场，并区分了 FRED / yfinance / akshare 来源；
- 板块轮动已支持 GICS / 申万一级双分类，且成份股构成 drill-down 已落地。

但从**职业宏观研究员**的视角看，当前四个模块仍存在三类核心短板：

1. **指标覆盖“看盘有余、研究不足”**：缺少信用利差、实际利率、经济领先指标、经济 surprise index、资金流等研究员日常跟踪的系列；
2. **口径与频率披露不充分**：同比/环比、季节调整/非季调、发布频率等关键元数据未在 UI 上显性标注；
3. **可视化专业性不够**：缺少 recession shading、同比/环比切换、多指标对比、期限结构、热力图动态刻度等宏观研究标配能力。

以下按模块列出问题清单与修复建议。

## 2. 宏观 `/macro` 模块

### 2.1 指标覆盖与分类

#### 问题 1：美国宏观指标缺少信用、实际利率、领先类指标

- **位置**：`app/services/macro/fred_service.py:60-101`（`SERIES_REGISTRY`）
- **问题描述**：当前 FRED 注册表约 30 个 US 系列，覆盖增长、通胀、就业、货币、利率、市场六大类的基础项，但缺少：
  - **信用利差**：BBB-10Y 利差、高收益利差（BAMLH0A0HYM2）、IG 利差等；
  - **实际利率 / 通胀预期**：10Y TIPS 实际收益率（DFII10）、5Y5Y 远期通胀预期（T5YIFR）、10Y 盈亏平衡通胀（T10YIE）；
  - **经济领先指标**：谘商会 LEI（USSLIND）、费城联储制造业（USPHCI）、纽约联储帝国制造业、零售销售控制组；
  - **房地产**：成屋销售、新屋销售（仅 HOUST 开工）、Case-Shiller 房价指数、30Y 抵押贷款利率（MORTGAGE30US）；
  - **劳动力质量**：JOLTS 职位空缺、平均时薪、劳动参与率、U6 失业率；
  - **国际贸易**：贸易差额、美元指数狭义（DTWEXAFEGS）已有，但缺少贸易加权广义汇率指数。
- **专业影响**：宏观研究员判断“经济周期位置”和“货币政策拐点”高度依赖信用利差和实际利率；缺少这些指标，无法判断金融条件是否收紧、通胀预期是否脱锚。
- **建议修复方案**：
  1. 在 `SERIES_REGISTRY` 新增上述系列，按“信用”“实际利率/通胀预期”“房地产”等新增 category；
  2. 在后端增加 `MacroIndicatorCategory` 枚举或元数据表，允许前端按 category 过滤；
  3. 提供“经济周期”预设视图（增长/通胀/就业/货币/信用/市场六宫格）。
- **优先级**：P1

#### 问题 2：欧元区指标过于单薄，且缺少关键高频指标

- **位置**：`app/services/macro/fred_service.py:138-147`（`_EU_SERIES`）
- **问题描述**：当前欧元区仅覆盖 GDP、CPI、失业率、欧央行存款便利利率 4 个系列，缺少：
  - 欧元区 PPI、核心 CPI（HICP ex energy/food）、零售销售、工业产出；
  - 制造业/服务业 PMI（HCOB/IHS Markit）、ZEW 经济景气指数、消费者信心指数；
  - 欧元区 M3、信贷脉冲、贸易差额；
  - 欧元区 10Y 国债收益率（BTP/德国国债利差即 spread）等。
- **专业影响**：仅 4 个低频指标无法判断欧元区经济动量和通胀粘性，无法与美联储政策进行跨央行比较。
- **建议修复方案**：
  1. 扩展 `_EU_SERIES` 至少覆盖 HICP 核心、PPI、零售销售、工业产出、制造业/服务业 PMI、M3、10Y 德债收益率；
  2. 如 FRED 数据点稀疏，可引入 ECB Statistical Data Warehouse（SDW）或 OECD 数据作为补充 source。
- **优先级**：P1

#### 问题 3：中国宏观指标覆盖面窄，且缺少货币、信贷、财政、贸易

- **位置**：`app/services/macro/scheduler.py:27-36`（akshare 宏观指标映射）
- **问题描述**：当前中国宏观仅 7 个系列：GDP/CPI/PPI/M2/PMI/SHIBOR/RRR。缺少：
  - **货币与信贷**：LPR、MLF、社会融资规模、人民币贷款、M1/M2 剪刀差、信贷脉冲；
  - **财政**：财政支出、财政收入、政府债券发行；
  - **贸易与投资**：进出口、贸易差额、固定资产投资、房地产投资、社会消费品零售总额；
  - **房地产**：70 城房价、商品房销售面积、新开工面积；
  - **劳动力**：城镇调查失业率、PMI 就业分项。
- **专业影响**：中国市场分析核心在于“信用周期”和“房地产周期”。没有社融、信贷脉冲、LPR、房地产数据，就无法判断中国经济是处在加杠杆还是去杠杆阶段。
- **建议修复方案**：
  1. 在 akshare provider 中新增 `fetch_china_macro_credit`、`fetch_china_macro_lpr`、`fetch_china_macro_trade`、`fetch_china_macro_fai`、`fetch_china_macro_realestate`；
  2. 增加 `cn` 区指标分类标签（增长、通胀、货币、信用、财政、房地产、贸易）。
- **优先级**：P0

#### 问题 4：中国 CPI 命名与口径存在误导

- **位置**：`app/data/providers/akshare_provider.py:532-556`（`fetch_china_macro_cpi`）
- **问题描述**：该方法从 `macro_china_cpi_monthly` 取“今值”，命名为 `CPI 月率`，但 code 为 `cpi_yoy`。在宏观研究中，
  - “CPI 月率”通常指月度环比（MoM）；
  - “CPI 同比”才是年度同比（YoY）。
  当前命名与代码不一致，容易让用户误以为是环比数据。
- **专业影响**：研究员需要区分同比（看趋势）和环比（看月度动量）。混淆会导致对通胀拐点的错误判断。
- **建议修复方案**：
  1. 将 `name_zh` 改为 `CPI 同比增长`（或根据实际列明确标注）；
  2. 若 akshare 提供环比数据，新增 `cpi_mom` 系列；
  3. 在 UI 中显式标注 `同比`/`环比` 标签。
- **优先级**：P0

#### 问题 5：指标分类未在 UI 上显性展示，无法按增长/通胀/就业等维度筛选

- **位置**：`web/src/pages/Macro/index.tsx:324-391`（指标表格列定义）
- **问题描述**：表格列仅含“指标、最新值、最新期、更新时间、操作”，没有 category、frequency、seasonal_adjustment 列；左侧也没有分类筛选器。
- **专业影响**：研究员日常看板不是按指标字母顺序，而是按“增长、通胀、就业、货币、信用、市场”六宫格快速定位。
- **建议修复方案**：
  1. 表格增加 `category` 和 `frequency` 列（或至少作为 tag 显示在指标名下方）；
  2. 顶部 Segmented 区域选择器之外，新增 category 过滤器（多选 Checkbox / Tag）；
  3. 后端在 `MacroIndicatorItem` schema 中返回 `category` 和 `frequency`。
- **优先级**：P1

#### 问题 6：未提供指标频率与季节调整信息

- **位置**：`app/schemas/macro.py:8-19`（`MacroIndicatorItem`）
- **问题描述**：schema 没有 `frequency`（daily/weekly/monthly/quarterly/annual）和 `seasonal_adjustment`（SA/NSA）字段。FRED 中 CPIAUCSL 是 SA，PPIACO 是 NSA，但用户无法从 UI 判断。
- **专业影响**：做跨指标对比时，必须知道频率是否一致、是否季节调整。例如直接用 NSA 的 PPI 和 SA 的 CPI 做相关性分析会失真。
- **建议修复方案**：
  1. 在 `MacroIndicator` 模型和 schema 中新增 `frequency` 和 `seasonal_adjustment` 字段；
  2. 在 `FredSeriesMeta` 注册表中维护该元数据；
  3. 前端指标名后增加 `SA`/`NSA` 小标签和 `M`/`Q`/`D` 频率标签。
- **优先级**：P0

### 2.2 图表与可视化

#### 问题 7：宏观图表缺少 NBER recession shading

- **位置**：`web/src/pages/Macro/index.tsx:393-420`（`chartOption`）
- **问题描述**：当前折线图只有基础网格、坐标轴和面积图，没有经济衰退期阴影背景。宏观研究中 recession shading 是判断周期拐点、资产相关性变化的标准工具。
- **专业影响**：无法直观看到指标在衰退前后如何变化，也无法判断当前读数处于历史周期什么位置。
- **建议修复方案**：
  1. 在后端维护一份 NBER 衰退区间表（可硬编码或放在 DB），通过 API 返回；
  2. 在 ECharts `markArea` 中渲染衰退区间为半透明灰色背景；
  3. 提供开关允许用户隐藏/显示 recession shading。
- **优先级**：P0

#### 问题 8：不支持同比/环比/绝对值切换

- **位置**：`web/src/pages/Macro/index.tsx:393-420`（`chartOption`）
- **问题描述**：对于 CPI、PPI、GDP 等价格指数，研究员通常需要：
  - 同比（YoY）变化率；
  - 环比（MoM）变化率；
  - 原始指数水平。
  当前图表只展示原始指数或数值，没有切换按钮。
- **专业影响**：CPI 指数水平长期趋势向上，直接看水平值难以判断通胀拐点；同比/环比才是交易和研究的核心。
- **建议修复方案**：
  1. 后端在 `/macro/indicators/{code}` 返回原始点的同时，提供可选的 `transform=yoy|mom|level`；
  2. 前端在图表工具栏增加 `原始值 / 同比 / 环比` Segmented 切换；
  3. 对于已经是增长率序列（如 GDP YoY），默认展示原始值。
- **优先级**：P1

#### 问题 9：图表时间轴和缩放能力不足

- **位置**：`web/src/pages/Macro/index.tsx:393-420`（`chartOption`）
- **问题描述**：`xAxis` 为 category 类型，且固定 limit=365。对于月频/季频数据，365 天只覆盖约 12-30 个点，无法看长期趋势；也没有区域缩放、时间范围选择器。
- **专业影响**：宏观分析通常需要 5-10 年乃至完整历史序列。当前设计更适合日内/日频看盘。
- **建议修复方案**：
  1. 将 `xAxis` 改为 `time` 类型（基于 `period` 时间戳），支持 ECharts `dataZoom`；
  2. 提供时间范围快捷按钮：`1Y / 5Y / 10Y / 全部`；
  3. 根据指标频率动态调整 limit（日频 365，月频 120，季频 40）。
- **优先级**：P1

#### 问题 10：图表未显示单位、来源和预期更新日期

- **位置**：`web/src/pages/Macro/index.tsx:393-420`（`chartOption`）
- **问题描述**：`yAxis.name` 仅显示单位，但没有数据来源、频率、下次发布日期等上下文。鼠标悬停 tooltip 也只有 period/value，没有同比/环比、前值、预期值（consensus）等。
- **专业影响**：宏观研究员看图表时高度依赖“最新值 vs 预期 vs 前值”的 surprise 结构。缺少这些，图表可读性大打折扣。
- **建议修复方案**：
  1. 在图表顶部增加来源、频率、最近更新、下次发布日期的信息条；
  2. Tooltip 增加“前值”、“变化率”字段；
  3. 长期引入共识预期（consensus）数据，展示 actual/estimate/ prior 三列。
- **优先级**：P1

#### 问题 11：不支持多指标同屏对比

- **位置**：`web/src/pages/Macro/index.tsx:555-600`（走势面板）
- **问题描述**：当前点击指标只能看单个指标的折线，无法把多个指标（如 CPI 和核心 CPI、10Y 收益率和实际收益率）放在同一坐标系对比。
- **专业影响**：宏观研究的核心是“关系”而非“单点”。例如判断实际利率是否压制风险资产，需要同时看名义利率、通胀预期和股市。
- **建议修复方案**：
  1. 在表格中增加多选复选框，用户可选 2-4 个指标对比；
  2. 支持双 Y 轴（左右轴），允许不同单位指标叠加；
  3. 提供“相关性对比”预设（如 CPI vs 核心 CPI、失业率 vs 初请）。
- **优先级**：P1

### 2.3 数据质量与交互

#### 问题 12：数据新鲜度提示仅覆盖 FRED，且文案过于简化

- **位置**：`app/services/macro_service.py:399-425`（`_compute_freshness_hint`）
- **问题描述**：当前 `freshness_hint` 只对 `source=fred` 且滞后 >1 天的行生成，文案固定为“FRED 数据为 H.10 周度发布”。该文案只适用于 FX 周度系列，不适用于月度 CPI、非农就业等。
- **专业影响**：用户看到“数据延迟”提示但不理解为什么延迟，也无法判断是正常发布节奏还是调度失败。
- **建议修复方案**：
  1. 在 `FredSeriesMeta` 中增加 `expected_lag_days` 和 `release_frequency` 元数据；
  2. 根据指标频率生成更精准的提示，例如“CPI 为月度数据，约延迟 15 天”；
  3. 对 akshare/yfinance 数据也进行 freshness check，尤其是中国月度数据。
- **优先级**：P1

#### 问题 13：缺少宏观数据导出与 API 说明

- **位置**：`web/src/pages/Macro/index.tsx:207-604`（整个页面）
- **问题描述**：页面没有导出 CSV/Excel 按钮，也没有 API 调用示例或 Python snippet。
- **专业影响**：研究员通常需要把数据导入 Python/R/Excel 做进一步建模、回测、回归分析。
- **建议修复方案**：
  1. 在表格工具栏增加“导出 CSV”按钮，调用 `/macro/indicators-list` 分页下载；
  2. 提供“复制 API 调用”按钮，自动生成 `curl` 或 Python 代码；
  3. 对 admin 用户提供后台刷新日志查看入口。
- **优先级**：P2

### 3.1 区域划分与覆盖度

#### 问题 14：/global 页面按资产类别分组，未按地理区域分组

- **位置**：`web/src/pages/GlobalMarkets/index.tsx:116-123`（`CATEGORY_LABELS` / `CATEGORY_ORDER`）
- **问题描述**：当前分组为“美债利率、外汇、大宗商品、主要指数、情绪/波动”，没有按“美洲、亚太、欧洲、其他新兴市场”区域分组。
- **专业影响**：研究员早上看全球市场时，习惯按区域顺序扫盘：亚太收盘 → 欧洲开盘 → 美洲收盘。当前分类适合看资产类别，但不适合按区域快速定位市场状态。
- **建议修复方案**：
  1. 在页面顶部保留资产类别分组，同时增加“区域视图”Tab（美洲/亚太/欧洲/EM）；
  2. 在 Dashboard 的 PULSE 区域已经有区域分组（`PULSE_GROUPS`，`us_equity`、`asia_pacific`、`europe`、`crypto`），建议复用该分组逻辑到 `/global`。
- **优先级**：P1

#### 问题 15：全球指数覆盖仍缺重要市场和资产

- **位置**：`web/src/pages/GlobalMarkets/index.tsx:150-191`（`PRIMARY_CODES`）和 `app/data/providers/yfinance_indices_provider.py:75-89`
- **问题描述**：当前已覆盖主要发达市场指数，但缺少：
  - 欧洲 STOXX 600、Euro Stoxx 50；
  - 新兴市场：MSCI EM、巴西 BOVESPA、墨西哥 IPC、南非 JSE；
  - 大中华区：恒生科技、上证 50、科创 50、创业板指；
  - 美股行业 ETF：XLF、XLK、XLE、XLU、XLP、XLI、XLB、XLRE、XLC；
  - 债券 ETF：TLT（20Y+ 美债）、IEF（7-10Y 美债）、LQD（投资级信用）、HYG（高收益信用）；
  - 商品：黄金、白银、铜、天然气、大豆/玉米（当前 Brent/WTI 已覆盖，但黄金被注释掉）。
- **专业影响**：全球资产配置和跨市场轮动需要更完整的市场 beta 和 sector exposure。缺少黄金、信用债、行业 ETF、新兴市场，无法做“风险-避险”和“周期-防御”切换分析。
- **建议修复方案**：
  1. 在 yfinance registry 中补充上述 ticker；
  2. 增加 `asset_class` 标签（equity/fixed_income/commodity/fx/crypto），并支持按资产类别过滤；
  3. 恢复黄金覆盖（通过 yfinance `GC=F` 或伦敦金替代 FRED 已废弃的 `GOLDAMGBD228NLBM`）。
- **优先级**：P1

#### 问题 16：美债收益率曲线缺少 2Y、5Y、10Y、30Y 完整期限结构

- **位置**：`web/src/pages/GlobalMarkets/index.tsx:150-157`（`PRIMARY_CODES` 中 rate 类）
- **问题描述**：当前 rate 类只有 30Y/10Y/2Y 收益率和两个利差（T10Y2Y、T10Y3M），没有 5Y 收益率，也没有把整条曲线以“期限结构图”形式展示。
- **专业影响**：收益率曲线形状（陡峭/倒挂/平坦）是判断衰退和银行利差的核心。研究员需要看整条曲线而不仅是 2Y-10Y spread。
- **建议修复方案**：
  1. 在 FRED registry 中增加 DGS5、DGS3M（已有 T10Y3M 但缺少单独 3M）和 DGS1M；

### 3.2 数据口径与显示

#### 问题 17：日涨跌口径不一致，容易引起误解

## 4. 板块轮动 `/sector-rotation` 模块

### 4.1 计算逻辑与口径

#### 问题 22：板块收益采用简单平均，未做市值/规模加权

- **位置**：`app/services/sector_rotation_service.py:626-628`（`avg_1m = avg(vals["return_1m"])`）
- **问题描述**：`analyze_sectors` 对每个 sector 内的所有 instrument 做等权平均，没有按市值（stock）或基金规模（ETF）加权。
- **专业影响**：等权平均会过度放大小盘股和主题 ETF 的影响，无法反映真实的板块市值表现。例如一个板块如果小盘股大涨但龙头滞涨，等权平均会显著高估板块强度。
- **建议修复方案**：
  1. 在聚合时按权重计算加权平均收益：
     - 个股用 `total_mv`（万元）转换后市值；
     - ETF 用 `fund_size`；
  2. 提供“等权”和“加权”两种视图切换；
  3. 加权作为默认，等权作为补充。
- **优先级**：P0

#### 问题 23：个股与 ETF 混合存在双重计数问题

- **位置**：`app/services/sector_rotation_service.py:60-61` 和 `561-604`（聚合逻辑）
- **问题描述**：板块内同时放入个股和 ETF（如半导体 ETF 和半导体股票），ETF 本身就包含底层股票，造成板块暴露被重复计算。
- **专业影响**：会导致板块收益和相对强度被系统性放大， especially for 主题板块；ETF 权重越高，重复计算越严重。
- **建议修复方案**：
  1. 提供两种视图：
     - “纯个股”：剔除 ETF，只看股票层面的行业表现；
     - “ETF 映射”：只看 ETF，作为行业 beta 的 proxy；
     - “混合（当前）”：保留现有逻辑，但明确标注存在双重计数风险；
  2. 在聚合前计算每只 ETF 的底层行业暴露（需要 holdings 数据），按暴露权重分配收益。
- **优先级**：P0

#### 问题 24：相对强弱计算为收益比值，而非标准 RS 指标

- **位置**：`app/services/sector_rotation_service.py:631-633`（`rs_1m = avg_1m / market_avg["return_1m"]`）
- **问题描述**：当前 `relative_strength_1m` = 板块平均收益 / 市场平均收益。这本质上是“相对收益比”，不是技术分析或量化中常用的相对强弱（price ratio / RS ratio）。
- **专业影响**：当市场平均收益为负时，板块收益比会出现符号翻转和极端值，例如市场 -5%、板块 -1%，RS=0.2 表示“跑输”而非“跑赢”，与直觉相反。标准 RS 通常定义为 price / benchmark price，或相对收益率差。
- **建议修复方案**：
  1. 同时提供两种指标：
     - 相对收益差：`sector_return - market_return`（直观，正负清晰）；
     - 标准化 RS：`sector_return / benchmark_return` 或 price ratio；
  2. 在 tooltip 和表头解释口径；
  3. 避免在 market_avg 为负时作为主要指标展示。
- **优先级**：P0

#### 问题 25：轮动信号仅依赖排名变化 ≥3 位，过于简单且噪音大

- **位置**：`app/services/sector_rotation_service.py:701-788`（`_detect_rotation_signals`）
- **问题描述**：当前轮动信号 = 板块在 1 月收益排名上较前一期上升/下降 ≥3 位。没有考虑：
  - 排名变化的持续性（连续 2-3 期上升才更可靠）；
  - 成交量/资金流确认；
  - 相对强弱的趋势方向；
  - 板块波动率差异。
- **专业影响**：单纯排名变化容易产生大量假信号，在震荡市中反复触发，导致追涨杀跌。
- **建议修复方案**：
  1. 引入更稳健的信号：
     - 排名连续两期变化 ≥3 位；
     - 相对强弱 1m / 3m 同时 >1；
     - 成交额放量（amount_total 增加）；
  2. 区分“动量增强”（momentum continuation）和“轮动进入”（rotation entry）两种信号；
  3. 提供信号历史回测面板，展示信号胜率。
- **优先级**：P1

### 4.2 覆盖范围与分类

#### 问题 26：板块轮动仅覆盖 A 股，缺少美股/港股/全球板块轮动

- **位置**：`app/services/sector_rotation_service.py:216-219`（`_SCOPE_MARKET = "A股"`）
- **问题描述**：Service 明确限定为 A 股（沪深北）ETF+STOCK，无法做美股行业轮动（GICS sector）或港股行业轮动。
- **专业影响**：宏观研究员通常需要同时观察 A 股、美股、港股的行业轮动，尤其是中美科技周期、消费周期、能源周期的错位。
- **建议修复方案**：
  1. 将 `_SCOPE_MARKET` 参数化，支持 `A股`、`US`、`HK`、`Global`；
  2. 美股数据可用 yfinance 的 11 个 GICS sector ETF（XLY、XLP 等）作为 proxy；
  3. 在 UI 顶部增加市场选择器（A股 / 美股 / 港股）。
- **优先级**：P1

#### 问题 27：ETF 行业映射依赖关键词启发式，准确性有限

- **位置**：`app/services/sector_rotation_service.py:83-141`（`ETF_SECTOR_HINTS`）
- **问题描述**：ETF 行业通过 sub_category/underlying_index 中的关键词匹配，例如含“科技”→ IT。这容易误判：
  - “科技创新” ETF 可能属于科创板宽基，不是纯科技；
  - “新能源”被映射到 Utilities，但实际可能跨行业；
  - “光伏”被映射到 IT 而非 Materials/Industrials。
- **专业影响**：板块分类错误会污染轮动信号，导致用户看到错误的领涨/领跌板块。
- **建议修复方案**：
  1. 引入更权威的分类源：Tushare/akshare 的 ETF 分类表、基金公司招募说明书、ETF holdings 映射；
  2. 对无法明确分类的 ETF，标记为“未分类”或“宽基”，不混入行业轮动；
  3. 提供分类置信度分数，低置信度项在 UI 中灰色展示。
- **优先级**：P1

### 4.3 可视化

#### 问题 28：热力图颜色刻度固定为 -6% 到 +6%，高波动期会截断

- **位置**：`web/src/pages/SectorRotation/index.tsx:255-267`（visualMap 配置）
- **问题描述**：`visualMap` 硬编码 `min=-6, max=6`。在牛市或熊市中，板块 1 年收益可能超过 50% 或低于 -30%，此时热力图颜色会全部顶到上限，失去区分度。
- **专业影响**：无法在高波动环境中快速识别真正的强势/弱势板块。
- **建议修复方案**：
  1. 根据当前数据动态计算 `min/max`（如取 ±1 倍标准差或 ±max(abs(value)) 的 95% 分位）；
  2. 对超出范围的颜色使用极值色；
  3. 提供按周期（1w/1m/3m/6m/1y）分别自适应的选项。
- **优先级**：P0

#### 问题 29：缺少板块轮动历史时间线和轮动矩阵

- **位置**：`web/src/pages/SectorRotation/index.tsx`（整体页面）
- **问题描述**：当前只有静态热力图、排名柱状图和相对强弱图，没有展示“过去 N 期板块排名变化”的时间线或轮动矩阵。
- **专业影响**：轮动研究的关键是持续性和路径。研究员需要看哪些板块在持续走强、哪些在反复横跳。
- **建议修复方案**：
  1. 新增“板块排名时间线”：x 轴为时间，y 轴为板块，用颜色/连线展示排名变化；
  2. 新增“轮动矩阵”：当前排名 vs 上期排名，快速定位上升/下降板块。
- **优先级**：P1

#### 问题 30：缺少板块估值和基本面数据

- **位置**：`app/services/sector_rotation_service.py`（整体设计）
- **问题描述**：板块轮动只基于价格和成交量指标，没有 P/E、P/B、盈利预期、资金流等行业基本面数据。
- **专业影响**：纯价格动量轮动容易买在估值高位。研究员通常需要结合“估值-盈利-动量”三因子做轮动。
- **建议修复方案**：
  1. 在 `ETFIndicator` 或新增 `sector_fundamental` 表中增加板块估值（P/E、P/B）和盈利预期；
  2. 提供“估值分位”和“盈利修正”列；
  3. 提供“动量-估值”散点图，帮助识别“高动量+低估值”机会。
- **优先级**：P2

- **建议修复方案**：
  1. 新增 `/macro/compare` 页面，支持选择 2-4 个指标；
  2. 显示双轴图、滚动相关系数、格兰杰因果提示（仅作为启发）；
  3. 提供常见预设（如 CPI vs 核心 CPI、失业率 vs 初请）。
- **优先级**：P2

#### 问题 38：术语词典对宏观指标解释偏基础，缺少研究级解读

- **位置**：`web/src/utils/termDictionary.ts:786-867`（宏观指标词条）
- **问题描述**：现有词典解释多为一句话定义，缺少“该指标与资产价格的关系”、“常见阈值”、“发布机构/频率”、“与其他指标的联动关系”等研究员关心的内容。
- **专业影响**：对于非专业用户，当前解释够用；但对研究员来说，需要更深入的解读框架。
- **建议修复方案**：
  1. 扩展宏观词条：增加 `release_agency`、`frequency`、`typical_threshold`、`asset_impact`、`related_indicators` 字段；
  2. 区分“新手模式”和“专业模式”的解释深度；
  3. 在 HelpPopover 中展示相关指标和常见组合。
- **优先级**：P2
