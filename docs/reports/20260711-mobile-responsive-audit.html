AD-Research 前端移动端适配问题审计报告

审计日期：2026-07-11
审计断点：375px（小屏手机）、768px（平板/小桌面）
严重度说明：P0 = 阻塞级，P1 = 重要，P2 = 优化
关键摘要
  • 共发现 2 个 P0、16 个 P1、16 个 P2 问题。
  • 最严重：AppLayout 主内容区在移动端未重置 margin-left: 240px，375px 下可用宽度仅约 103px。
  • Login 登录面板在 375px 下因 content-box 内边距溢出视口 64px，输入框被截断。
  • 高频问题：大量 size="small" 按钮仅 32px 高、管理表格列宽过宽、768px 断点未充分覆盖。

审计范围

  • web/src/components/AppLayout.tsx
  • web/src/pages/Login.tsx
  • web/src/pages/AdminUsers/index.tsx
  • web/src/pages/AdminDeployments/index.tsx
  • web/src/pages/ETLOpsDashboard/index.tsx
  • web/src/pages/NotificationConfig/index.tsx
  • 相关样式：web/src/styles/global.css、web/src/components/AppLayout.css 等

P0 阻塞级问题

P0

1. AppLayout — 移动端主内容区未重置 margin-left: 240px

文件：AppLayout.tsx 第 596 行；styles/global.css 第 3088-3095 行

问题：在 <768px 时桌面侧边栏被移除，但 .app-layout__main 仍保持 margin-left: 240px，且 app-layout__main--mobile 无 CSS 重置。375px 视口下主内容区可用宽度仅剩约 103px，页面无法阅读。

修复：在移动端媒体查询中重置：

@media (max-width: 767px) {
  .app-layout__main { margin-left: 0; }
}
P0

2. Login — 375px 下登录面板因 content-box 内边距溢出截断

文件：styles/global.css 第 6953-6962 行、第 7254-7268 行

问题：.login-glass 使用 padding: 32px，面板使用 width: 100%，但未设置 box-sizing: border-box。渲染总宽 399px，超出 375px 视口 64px，导致输入框被截断。

修复：为面板添加 box-sizing: border-box，或改用 max-width: calc(100vw - 40px)。

P1 重要问题

P1

3. AppLayout — 主题切换按钮在移动端设置菜单中被隐藏

文件：AppLayout.css 第 84-89 行；AppLayout.tsx 第 361-376、699 行

问题：ThemeToggle 复用了 .app-layout__header-collapse 类，该类在移动端被 display: none 隐藏，导致移动端无法切换主题。

修复：为 ThemeToggle 单独定义类名，或移除该媒体查询。

P1

4. AppLayout — 移动端抽屉导航项触控目标不足

文件：styles/global.css 第 8519-8521 行、第 8523-8528 行

问题：抽屉分组标题实际高度约 32-36px，二级导航项 min-height: 40px，均低于 44px 推荐触控目标。

修复：统一设为 min-height: 44px，上下 padding 10-12px。

P1

5. AppLayout — 768px 平板侧边栏触控目标仍偏小

文件：styles/global.css 第 2834-2852 行、第 2926-2954 行、第 8171-8461 行

问题：768px 断点显示桌面侧边栏，但导航项默认仅 36px，分组标题约 36px，collapsed 图标项 40px，均未达 44px。

修复：在 max-width: 991px 下统一设置 min-height: 44px。

P1

6. AppLayout — 使用 100vh 而非 100dvh

文件：styles/global.css 第 6-10 行、第 2764 行

问题：min-height: 100vh 在 iOS Safari 等移动浏览器中会因动态工具栏导致 footer 被遮挡或出现不可点击区域。

修复：

min-height: 100vh;
min-height: 100dvh;
P1

7. AppLayout — 固定元素未处理安全区

文件：styles/global.css 第 3102-3115 行（header）、第 3214-3222 行（footer）、第 8510 行（drawer logo）

问题：header、footer、抽屉 logo 未使用 env(safe-area-inset-top/bottom)，在刘海屏/灵动岛上可能被系统 UI 遮挡。

修复：viewport meta 添加 viewport-fit=cover；header 加 padding-top: env(safe-area-inset-top, 0)；footer 加 padding-bottom: env(safe-area-inset-bottom, 0)。

P1

8. AppLayout — 768px 平板 header 右侧控件拥挤

文件：AppLayout.tsx 第 663-669 行

问题：768px 且侧边栏展开时，header 右侧同时渲染 CollapseToggle、DensityToggle、ColorConventionToggle、ThemeToggle，长面包屑被挤压。

修复：在 768px-991px 将 DensityToggle/ColorConventionToggle 移入 settings dropdown，或给 header-controls 添加 flex-wrap。

P1

9. Login — 输入框 14px 字体会触发 iOS 缩放

文件：styles/global.css 第 7139-7148 行

问题：.login-input 字号 14px，iOS 聚焦时自动缩放。

修复：移动端改为 font-size: 16px。

P1

10. Login — 100vh + overflow:hidden 导致小屏/横屏内容截断

文件：styles/global.css 第 6574-6582 行

问题：.login-page 同时设置 height: 100vh 与 overflow: hidden。在 iPhone SE 横屏等场景下内容超出 100vh 且无法滚动。

修复：

.login-page { height: auto; min-height: 100vh; overflow-y: auto; }
P1

11. Login — 768px 断点未覆盖，双列布局品牌名溢出

文件：styles/global.css 第 6943-6950 行、第 6999-7028 行、第 7247 行

问题：断点为 max-width: 767px，在 768px 仍走双列布局，品牌名被压缩溢出。

修复：将媒体查询改为 max-width: 768px 或 991px。

P1

12. AdminUsers — 操作列 240px 过宽且按钮未堆叠

文件：pages/AdminUsers/index.tsx 第 157-196 行

问题：操作列固定 240px，三个按钮并排，375px 下占约 2/3 屏幕宽度。

修复：移动端将操作改为下拉菜单或垂直堆叠按钮；降低操作列固定宽度。

P1

13. AdminUsers — 操作按钮高度仅 32px

文件：pages/AdminUsers/index.tsx 第 161-195 行

问题：按钮使用 size="small"，高度 32px，低于 44px 触控目标。

修复：移动端改用 size="middle" 或媒体查询强制 min-height: 44px。

P1

14. AdminDeployments — 部署历史表格 7 列合计过宽

文件：pages/AdminDeployments/index.tsx 第 63-156 行

问题：7 列固定宽度合计约 740px，在 375px/768px 下都远超视口。

修复：小屏隐藏次要列（分支、耗时、触发人），或改用卡片列表。

P1

15. AdminDeployments — Commit SHA 未截断撑开列宽

文件：pages/AdminDeployments/index.tsx 第 97-113 行

问题：显示完整 40 位 SHA，未做截断。

修复：截断为 7-8 位并加 Tooltip。

P1

16. ETLOpsDashboard — Descriptions 组件强制两列布局

文件：pages/ETLOpsDashboard/index.tsx 第 147 行

问题：Descriptions 使用 column={2}，375px 下每列仅约 150px，标签与内容拥挤。

修复：改为响应式列数 column={{ xs: 1, sm: 1, md: 2, lg: 2 }}。

P1

17. ETLOpsDashboard — freshnessBadge 长文本溢出

文件：pages/ETLOpsDashboard/index.tsx 第 156-175 行

问题：日期 + “(<N>d 前)” 字符串过长，在两列布局中极易撑满单元格。

修复：移动端精简显示，或给 Badge 文本设置 max-width + 省略号。

P1

18. NotificationConfig — 通知配置表格固定列宽过宽

文件：pages/NotificationConfig/index.tsx 第 92-152 行

问题：列宽合计已达 550px，加上“详情”列，375px 下必须大幅横向滚动。

修复：小屏降低列宽或隐藏“详情”列，改为点击行展开。

跨页面观察与建议

  • 表格横向滚动均限制在容器内：所有表格都通过 scroll={{ x: 'max-content' }} 或 overflow-x: auto 将横向滚动限制在表格内部，未产生页面级横向滚动，符合响应式基本要求。但表格内部滚动距离过长，仍显著影响移动端可用性。
  • size="small" 按钮触控目标不足：AdminUsers、AdminDeployments、NotificationConfig 等页面大量使用 32px 高的小按钮，建议建立移动端按钮规范：768px 以下统一使用 size="middle" 或媒体查询强制 min-height: 44px。
  • 768px 断点覆盖不足：Login、AppLayout 等页面使用 max-width: 767px，导致 768px 平板仍走桌面布局。建议统一使用 max-width: 768px 或 991px。
  • Modal 宽度固定：AdminUsers 和 NotificationConfig 的 Modal 使用固定宽度，建议改为响应式宽度对象或 Drawer。
  • 文本截断策略缺失：长用户名、SHA、URL、邮箱、错误信息、任务 ID 等均未做截断，是表格横向滚动过长的主要原因之一。建议统一建立 ellipsis + Tooltip 策略。
