## 一、项目概述

### 1.1 目标

将手工化的 ETF 投研工作流升级为**一体化 Web 投研平台**，覆盖：

- 全市场 ETF 基础信息与行情数据自动化采集
- 技术指标与风险指标批量计算
- ETF 综合评分与多维度排名
- 标的池创建、权重配置、相关性分析与再平衡提示
- 策略配置、回测验证、交易信号生成与绩效归因
- 板块/行业轮动分析
- 自动化 HTML/Markdown 研究报告生成
- 可视化 Web 仪表盘与通知推送

### 1.2 业务范围

| 子系统 | 说明 |
|--------|------|
| 数据基础设施 | 数据采集、清洗校验、存储、指标计算、REST API |
| 投研核心 | 评分、筛选、标的池增强、报告生成 |
| Web 仪表盘 | React 前端可视化、交互式分析 |
| 策略系统 | 策略配置、回测引擎、信号生成、绩效归因 |

### 1.3 用户角色

平台内置 5 个账号（`AuthSettings.USERS` + `users` 表），由 JWT 认证保护：

| 角色 | 说明 |
|------|------|
| admin | 管理员 |
| Aidan / Tee / Zack / Philip | 普通投研用户 |

## 四、数据管道与 ETL

### 4.1 数据源抽象

`app/data/providers/base.py` 定义 `DataProvider` 抽象接口：

| 方法 | 说明 |
|------|------|
| `fetch_etf_list()` | 获取 ETF 基础信息列表 |
| `fetch_daily_bars(codes, start, end)` | 批量获取日 K 线 |
| `fetch_realtime_quotes(codes)` | 获取实时行情 |
| `get_market_hours(code)` | 获取交易时段 |
| `check_health()` | 健康检查 |

### 4.2 AkshareProvider（A 股）

文件：`app/data/providers/akshare_provider.py`

- **数据源**：默认东方财富 `fund_etf_hist_em`，连续失败 2 次后回退到新浪财经 `fund_etf_hist_sina`。
- **并发控制**：5 线程线程池，单 API 调用延迟 0.3s，批次间隔 2.0s。
- **重试机制**：最大 3 次重试。
- **代码规则**：代码以 `5` 开头后缀 `.SH`，以 `1/0` 开头后缀 `.SZ`。
- **返回字段**：`etf_code`, `trade_date`, `open`, `high`, `low`, `close`, `volume`, `amount`, `change_pct`, `turnover_rate`。

### 4.3 YFinanceProvider（跨境）

文件：`app/data/providers/yfinance_provider.py`

- **支持市场**：US、JP、HK。
- **代码映射**：
  - `.US` → 去除后缀（如 `SPY.US` → `SPY`）
  - `.JP` → `.T`（如 `1321.JP` → `1321.T`）
  - `.HK` → 保留 `.HK`
- **日线获取**：逐个 ticker 调用 `yf.Ticker.history()`，`amount` 由 `volume * close` 估算。
- **实时行情**：`yf.download(period="1d", interval="1d")`。
- **外汇**：通过 `USDCNY=X` 等货币对获取汇率。

### 4.4 ETL 流程

`app/data/pipelines/base.py` 定义 `ETLPipeline` 抽象：

```
extract() 获取原始数据
    ↓
normalize() 列名/类型标准化
    ↓
validate_all() 四层校验
    ↓
load() 写入数据库（UPSERT）
    ↓
post_process() 后处理/缓存失效
```

### 4.5 四层数据校验

`app/data/transformers/validator.py`

| 层级 | 名称 | 检查内容 | 是否阻断 |
|------|------|---------|---------|
| L1 | 格式校验 | 必填列、缺失值 | 是 |
| L2 | 业务校验 | `high >= low`、`close` 在 `[low, high]` 内、成交量非负 | 是 |
| L3 | 时序校验 | 涨跌幅绝对值是否异常（>20% 仅警告） | 否 |
| L4 | 完整性校验 | 预期 ETF 是否有缺失 | 否 |

### 4.6 A 股日终 ETL

`app/data/pipelines/a_share.py`

- 作业名：`a_share_daily_etl`
- 默认采集前一日数据，向前取 7 天窗口覆盖节假日。
- 仅保留目标交易日数据。
- 使用 PostgreSQL `INSERT ... ON CONFLICT DO UPDATE` 实现 upsert。
- 完成后清除缓存：`indicator:*`、`screen:*`、`etf:list:*`。

### 4.7 数据标准化

`app/data/transformers/normalizer.py`

- 列名映射：兼容中文列名（日期、开盘、收盘、最高、最低、成交量、成交额、涨跌幅、换手率）与英文列名。
- 类型转换：`trade_date` → date，价格/金额 → numeric，`volume` → Int64，涨跌幅 → numeric。

## 五、指标计算体系

### 5.1 技术指标

`app/data/indicators/technical.py`

| 指标 | 公式/说明 |
|------|----------|
| MA5/10/20/60 | 简单移动平均 |
| RSI14 | Wilder 平滑 RSI，`RS = avg_gain / avg_loss`，`RSI = 100 - 100/(1+RS)` |
| MACD | DIF = EMA12 - EMA26；DEA = EMA9(DIF)；Histogram = DIF - DEA |
| ATR14 | True Range 滚动 14 日均值 |
| 布林带 | MA20 ± 2 × std20 |

### 5.2 风险与收益指标

`app/data/indicators/risk.py`

| 指标 | 窗口 | 说明 |
|------|------|------|
| `volatility_20d` | 20 日 | 年化波动率，`std(returns) * sqrt(252) * 100` |
| `volatility_60d` | 60 日 | 同上 |
| `max_drawdown_1y` | 252 日 | `(prices - cummax) / cummax * 100` |
| `sharpe_1y` | 252 日 | `(mean(return)*252 - 2%) / (std*sqrt(252))` |
| `return_1w` | 5 日 | 近一周收益 |
| `return_1m` | 21 日 | 近一月收益 |
| `return_3m` | 63 日 | 近三月收益 |
| `return_6m` | 126 日 | 近半年收益 |
| `return_1y` | 252 日 | 近一年收益 |

### 5.3 批量指标计算

`app/data/indicators/calculator.py`

- `batch_calculate_indicators(db, target_date=None, full_history=False)`
- 遍历所有 active ETF，读取历史日线，计算技术/风险指标。
- `full_history=True` 时重算全部历史；否则仅计算最新日期。
- 结果通过 upsert 写入 `etf_indicator` 表。
- 写入 `ETLLog` 记录执行状态（success / partial）。

## 六、综合评分系统

### 6.1 评分维度

`app/services/scoring_service.py` 定义 5 个维度：

| 维度 | 默认权重 | 关联指标 | 方向 |
|------|---------|---------|------|
| 收益能力 | 30% | `return_1m`, `return_3m`, `return_1y` | 越高越好（desc） |
| 风险控制 | 25% | `volatility_20d`, `max_drawdown_1y` | 越低越好（asc） |
| 夏普比率 | 25% | `sharpe_1y` | 越高越好（desc） |
| 流动性 | 10% | `amount`（日均成交额） | 越高越好（desc） |
| 趋势强度 | 10% | `rsi14`, `ma_position` | 越高越好（desc） |

### 6.2 内置评分模板

| 模板 | 特点 |
|------|------|
| 保守型 | 风险权重更高，收益权重更低 |
| 均衡型 | 收益、风险、夏普均衡 |
| 进取型 | 收益权重更高，风险容忍更高 |

### 6.3 计分逻辑

`app/data/indicators/scoring.py`

1. 对每个维度内的多个指标取平均。
2. 使用 `scipy.stats.rankdata` 计算百分位得分（0-100）。
3. `direction="asc"` 时反转百分位（用于风险类指标，越低越好）。
4. 综合得分 = Σ(维度得分 × 维度权重)。
5. 按综合得分降序排列，生成 1-based 排名。

### 6.4 排名输出

- `rank_overall`：全市场排名
- `rank_category`：同分类内排名

### 8.2 权重管理

`app/services/pool_enhancement_service.py`

| 功能 | 说明 |
|------|------|
| 目标权重 | 用户可设置每只 ETF 的目标权重，范围 0-100%，池内总和 ≤100% |

### 8.3 建议权重算法

| 算法 | 逻辑 |
|------|------|
| 等权（equal） | 100% / 成员数 |
| 评分加权（score） | 按综合评分占比分配 |
| 风险平价（risk_parity） | 按波动率倒数加权，低波动品种权重更高 |

- 成员列表与最新指标

### 8.5 相关性矩阵

- 取最近 60 日收盘价计算日收益。

## 九、策略、回测与信号

### 9.1 策略模板

`app/services/strategy_service.py`

| 策略类型 | 参数 |
|----------|------|
| 动量（momentum） | `window`（动量窗口）, `threshold`（阈值） |
| 均值回归（mean_reversion） | `z_score_threshold` |
| RSI（rsi） | `overbought`, `oversold`, `window` |

### 9.2 信号生成

`app/services/signal_generator.py`

- 读取 `ETFDailyBar` 最近 60+ 日数据。
- 根据策略类型计算信号：
  - **动量**：收益率超过阈值 → BUY；低于负阈值 → SELL。
  - **均值回归**：Z-score 超过正阈值 → SELL；低于负阈值 → BUY。
  - **RSI**：超买 → SELL；超卖 → BUY。
- 返回 `BUY / SELL / HOLD` 及信号强度。

### 9.3 回测引擎

`app/services/backtest_engine.py`

| 配置项 | 默认值 |
|--------|--------|
| 初始资金 | 100,000 |
| 佣金率 | 0.001 |
| 滑点率 | 0.001 |
| 仓位比例 | 1.0（满仓） |

**输出指标**：

- 总收益、年化收益、最大回撤、夏普比率
- 胜率、平均盈利/亏损、交易次数
- 每日 NAV 曲线、交易清单、信号序列

### 9.4 绩效归因

`app/services/attribution_service.py`

- 计算策略相对买入持有基准的超额收益。
- 采用简化 Brinson 模型：
  - **配置效应（Allocation Effect）**：择时贡献
  - **选择效应（Selection Effect）**：选股贡献
  - **交互效应（Interaction Effect）**：残差

### 9.5 策略对比

`app/services/strategy_comparison_service.py`

- 接收多个 `backtest_id`。
- 对比总收益、年化收益、最大回撤、夏普、胜率、交易次数。

## 十、板块轮动分析

`app/services/sector_rotation_service.py`

### 10.1 计算逻辑

1. 按 `category` 分组聚合 ETF 指标。
2. 计算板块平均收益（1 月 / 3 月）、夏普、波动率、RSI。
3. 计算相对强弱 `relative_strength` = 板块收益 - 全市场平均收益。
4. 按动量排序生成 `momentum_rank`。

### 10.2 轮动信号

- 对比当前排名与上一周排名。
- 排名上升 ≥3 位：生成 `up` 信号。
- 排名下降 ≥3 位：生成 `down` 信号。

## 十一、报告生成引擎

`app/services/report_service.py`

### 11.1 报告类型

| 类型 | 频率 | 内容 |
|------|------|------|
| 池周报（pool_weekly） | 每周 | 池概览、收益分析、风险分析、综合评分 |
| 池分析（pool_analysis） | 按需 | 同上 |

### 11.2 输出格式

- **HTML**：使用 Jinja2 模板渲染，内联 CSS，可直接浏览器打开。
- **Markdown**：表格形式，便于存档与版本管理。
- **JSON**：供前端消费。

### 11.3 HTML 报告模块

1. 头部：池名称、报告日期、成员数量
2. 概览：涨跌数量、平均综合得分、实时快照表
3. 收益分析：1 周 / 1 月 / 3 月 / 半年 / 1 年收益
4. 风险分析：20 日波动率、最大回撤、夏普、风险等级标签
5. 综合评分：排名表、维度得分条、金/银/铜等级
6. 页脚：生成时间戳

### 11.4 报告元数据

- 生成时先写入 `report_metadata` 状态为 `running`。
- 完成后更新 `status=done`、`file_path`、`file_size`。
- 失败时记录 `error_msg`。
