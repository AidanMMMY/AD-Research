一、设计原则

核心原则

原则说明
本地优先打开即有数据，网络请求不阻塞 UI
手势优先用原生手势替代 Web 端按钮操作
信息分层重要信息前置，详情按需展开
触控友好44pt 最小热区，不留死角
统一主题与 Web 端 Swiss Minimal + Neon Cyan 对齐

与 Web 端的关键差异

维度Web 端iOS App
导航固定侧边栏 + 页面跳转底部 TabBar + NavigationStack push
信息密度大屏，表格为主小屏，卡片 + 列表为主
交互点击、hover、右键菜单点击、长按、滑动、捏合
操作多按钮并列关键操作突出，次要收进更多菜单
刷新页面加载下拉刷新 / 自动刷新

四、iOS 原生交互规范

导航手势

手势场景反馈
右滑返回所有 push 页面跟随手指动画
下拉列表刷新RefreshControl 旋转
捏合K线缩放实时缩放
长按列表/图表Context Menu + Haptic
左滑列表项操作按钮（删除/收藏）

Haptic 触觉反馈

场景反馈类型
登录成功.notification(.success)
收藏/取消.impact(.light)
下拉刷新完成.impact(.medium)
删除确认.notification(.warning)

六、Widget 与系统集成

  • 桌面 Widget：Small/Medium 尺寸，显示自选标的价格
  • Live Activity：灵动岛实时价格（5秒更新）
  • Siri 捷径："打开我的标的池"
  • Apple Watch：complications 显示自选

七、无障碍设计

  • VoiceOver：所有按钮、标签、图表有 accessibilityLabel
  • 动态字体：支持系统 Dynamic Type
  • 减少动效：尊重系统设置
  • 色彩对比：Dark Mode 下 ≥ 4.5:1
  • 触控区域：最小 44x44pt
