一、设计原则

1.1 核心原则

原则 说明
本地优先打开即有数据，网络请求不阻塞 UI
手势优先用原生手势替代 Web 端按钮操作
信息分层重要信息前置，详情按需展开
触控友好44pt 最小热区，不留死角
统一主题与 Web 端设计系统对齐（⚠️ Web 端 2026-07-11 起已升级为 v2：蓝靛主色 #2563EB、暗色 #0D1117/#60A5FA；本文写成时的 "Swiss Minimal + Neon Cyan" 已过时）

1.2 与 Web 端的关键差异

维度 Web 端 iOS App
导航固定侧边栏 + 页面跳转底部 TabBar + NavigationStack push
信息密度大屏，表格为主小屏，卡片 + 列表为主
交互点击、hover、右键菜单点击、长按、滑动、捏合
操作多按钮并列关键操作突出，次要收进更多菜单
刷新页面加载下拉刷新 / 自动刷新
状态提示悬浮 tooltipHaptic 震动 + Toast

八、MVP 范围 vs 完整版

MVP（8周实现）

  • ✅ 4 个 Tab + K线详情 + AI 聊天 + 池子
  • ✅ 登录/登出/个人页面
  • ❌ Widget / Live Activity
  • ❌ 全市场筛选器
  • ❌ 回测创建
  • ❌ 策略管理
  • ❌ 相关性分析（受小屏限制）

完整版（MVP 后追加）

  • Widget + Live Activity + 后台刷新
  • 全市场筛选器（简化版）
  • 回测管理（只展示，输入放 Web 端）
  • 板块轮动卡片
  • Apple Watch complications
  • Siri 捷径
  • 多语言（英文/简体中文）
