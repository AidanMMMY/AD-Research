一、总体决策

决策结论
平台iOS 原生（Swift + SwiftUI/UIKit 混合）
底座复用现有 FastAPI 后端，API 直接调用
开发模式MVVM + Combine + async/await
图表DGCharts（K线）+ Swift Charts（辅助） — 免费
实时行情SSE 推送 — 1-5秒刷新
本地数据库GRDB — SQL 友好、高性能

首发市场香港区或美国区 App Store

二、技术选型

层级推荐方案选型理由
UI 框架SwiftUI 主体 + UIKitRepresentable 包裹图表新页面 SwiftUI 开发快；图表需 UIKit 性能
架构模式MVVM + CombineCombine 适合响应式数据流
网络层Alamofire + Codable拦截器、重试机制成熟
本地缓存GRDB金融查询复杂，SQL 更直接
认证存储KeychainJWT 必须安全存储
K线图表DGChartsApache 2.0 免费、散户级够用
辅助图表Swift Charts系统原生，收益曲线/饼图
图片加载Kingfisher列表缩略图缓存
推送APNs 原生系统级推送

四、认证与安全

改造后的认证接口

端点说明
POST /auth/login返回 access_token（15min）+ refresh_token（30天）
POST /auth/refresh刷新短期 token
POST /auth/logout置失效 token（Redis 黑名单）
POST /auth/device注册设备
GET /auth/devices查看已登录设备

Token 安全

  • access_token 存内存，refresh_token 存 Keychain
  • AuthInterceptor 自动注入 Bearer Token，401 自动刷新
  • 支持多端登录、设备解绑

五、图表方案

K线：DGCharts

  • Apache 2.0 免费许可
  • GitHub 27k+ stars
  • 支持捏合缩放、滑动、十字光标
  • 内置 MA/EMA/MACD/RSI/Bollinger
  • 需用 UIViewRepresentable 包进 SwiftUI

辅助图表：Swift Charts

  • 收益曲线、饼图用系统原生
  • 声明式 API，与 SwiftUI 配合佳
  • 热力图/雷达图用 DGCharts 补充

七、数据同步策略

核心原则

  1. 本地优先展示 — 打开 App 永远有数据
  2. 后台静默刷新 — 不阻塞用户操作
  3. 乐观更新 — 收藏/编辑等操作立即反馈
  4. 自动重试 — 网络恢复后自动同步 pending
  5. 冲突用时间戳 — 最后修改时间胜出
  6. 定期清理 — 防止数据库膨胀

八、商业化与 App Store

订阅策略

等级价格功能
免费版¥0基础行情、标的列表、3条评分
Pro 版¥18-38/月全部评分、高级筛选、回测、AI 笔记
机构版联系销售API 接入、自定义策略、白标

审核关键

  • 产品定位："数据分析工具"，不是"投资建议"
  • 所有页面标注"仅供参考，不构成投资建议"
  • AI 生成内容标注"AI 生成，需人工复核"
  • 不承诺任何收益率

上线路径

  1. TestFlight 内测（1-3 个月）
  2. App Store 香港/美国区上架
  3. 增加内购订阅
  4. 中国大陆区上架（视备案进展）
