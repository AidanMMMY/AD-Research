2026-07-17 /data 磁盘满导致 Redis 写失败、健康检查 degraded

关联文档：20260717-deploy-health-celery-followup、20260712-celery-worker-runbook

1. 现象

  • 时间：2026-07-17 约 12:46 UTC 部署成功（GitHub Actions #273）后，过了数小时。
  • 外部访问 https://www.alloyresearch.net/health 返回 degraded，db 与 redis 均为 error。
  • docker compose ps 显示 backend 与 postgres 均为 unhealthy，redis 健康但写操作被禁用。

2. 根因

  • ECS 数据盘 /dev/vdb（挂载 /data）100% 满。
  • Redis 启用 RDB + AOF 持久化，RDB 后台保存因 No space left on device 失败，触发 stop-writes-on-bgsave-error，写入命令全部返回 MISCONF 错误。
  • bankend /health 会尝试写 Redis（锁/计数器），命中写错误后报 ResponseError；DB 同样因磁盘满出现 OperationalError。

磁盘满构成（清理前）：

项目大小
Docker Images7.9 GB
Docker Build Cache7.3 GB
Docker Local Volumes21.5 GB
其中 aliyun-ecs_postgres_data11.9 GB
其中 aliyun-ecs_cninfo_pdfs8.0 GB

3. 修复过程

  1. 检查磁盘使用：df -h 确认 /data 100%。
  2. 清理 Docker 未使用镜像与构建缓存：
    docker image prune -af
docker builder prune -af
    共释放 4.817 GB。
  3. 清理后 /data 剩余 4.2 GB，Redis 恢复写能力，Postgres 健康检查恢复。
  4. 重新验证 /health：status=ok、ready=true、git_sha=0985d56。

4. 当前状态

  • /data 使用率从 100% 降至 89%，可用 4.2 GB。
  • 所有容器 healthy：backend、celery-worker、postgres、redis、nginx。
  • 队列正常消费，cninfo PDF 回填继续。
