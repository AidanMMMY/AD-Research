顶尖投资者如何发现机会 — 蒸馏为 AD-Research 平台功能

日期： 2026-07-02
作者： Claude（战略输入文档，非代码）

最后核实更新：2026-07-21（LLM 相关描述已更新为现状：平台默认 MiniMax、DeepSeek 为 legacy；功能/路线图建议未变）

目标读者： AD-Research 平台主理人
核心问题： 优秀的投资者怎么找到 idea？我们能把他们的方法蒸馏成平台功能吗？

0. 引言：为什么是这份文档

AD-Research 已经从"手工 ETF 投研"演进为"多市场 + 评分 + 池 + 回测 + 信号 + AI 研报"的 Web 投研平台，能力栈接近 Wind / Portfolio123 的入门级。但用户（您）在 2026-07-01 平台审计后指出的方向很清楚：

平台要继续演化为 "投资工作流"，而不仅仅是 "数据显示"。

这就引出一个根本问题：伟大的投资者是如何发现机会的？ 他们的方法学里有哪些"非显而易见"的步骤、哪些"反共识"的判断节点、哪些"二阶效应"的关注点？这些方法学中，哪些能编码成平台功能？哪些是平台永远无法替代的？

本报告分三段：

  1. 第一章：5+1 个投资者原型调研 — 跨价值、复合、宏观、事件驱动、量化、主题、特殊情境、A 股本土实务
  2. 第二章：平台功能地图 — 14 个具体功能点，附 UI 形态、数据依赖、复杂度、价值
  3. 第三章：实施路线图 + 工作流模板 — 按阶段落地的清单 + 每个原型的"如果我是 XX 投资者，今天我点哪里"

1. 投资者原型调研

方法论说明：每个原型不是"画像"，而是"工作流解剖"。重点是每天早上 8:30 这个人打开什么、为什么打开、看完之后做什么动作。浅层的"读 10-K 找护城河"不在本文档范围。

1.1 复合型价值 / Quality Compounder — Terry Smith, Nick Sleep, Tom Russo

原型画像
"买入可以持有 10 年的高质量生意"。拒绝轮动、拒绝择时、拒绝 90% 的标的池。决策频率极低（Smith 公开组合年周转 < 10%），研究深度极深。

早晨先看什么

  • 既有持仓的官方公告 / 监管文件（FCA filings / 港交所披露 / SEC EDGAR alerts）。不看行情终端，不看新闻推送。
  • 持仓公司一手经营数据（门店客流、APP DAU、信用卡刷卡量）— 他们用一手数据来"算"持仓的内在价值是否仍合理
  • "do nothing" list — 已研究的"贵但有道理"的公司（Smith 的 Nomad 清单），等估值杀下来再动手

筛选条件 / Anomalies

  • 高 ROIC（> 20% 持续 10 年）+ 净资产自由现金流为正 + 负债低（Net Debt / EBITDA < 1.5x）
  • "Boring" — 业务十年没大变化、行业增速 3-5%、管理层稳定 5+ 年
  • "不可分配支出" 占比低（Nick Sleep 的关键概念：把维护性 capex / 折旧加回去后，看是否仍能产生大量 FCF）
  • 复利秘密：再投资回报率（ROIC）显著高于资本成本，且增量资本仍能保持该水平
  • 估值上接受"看似贵"：用 owner earnings yield 而非 PE，对长期复合给予溢价

深度工作流（拿到候选后的 2-5 步）

  1. 写"墓志铭"：用一句话写清 10 年后这家公司会变成什么样，反向检查现状
  2. 算"反通胀" ：在通胀 5% 假设下，FCF 实际增长是否仍能支持买入价
  3. 看管理层资本配置历史：回购？分红？并购？并购后的整合记录？
  4. 找"理由 3 个"：列出 3 个会让你卖出的理由，看是否真实
  5. 30 天冷静期：写完不进池，30 天后再回看，识别情绪因素

买入触发（pull the trigger）

  • 估值出现"非系统性"下跌（市场情绪、行业冷风、汇率），但经营基本面没变
  • 出现"事件性套利"窗口：分拆、剥离、大股东减持引发的技术性折价
  • 多年跟踪后终于跌入"再投资回报率 - 资本成本"的合理折现区间

卖出信号

  • 增量 ROIC 跌到接近资本成本（"生意变平庸"）
  • 管理层资本配置明显变差（开始大额低回报并购 / 异常股权激励）
  • 估值进入"hot money zone"：出现散户、媒体集中推荐，机构持仓过度拥挤
  • 出现比它更便宜的同质替代品

对平台的 3-5 个可编码信号

  1. "高质量 + 合理价"综合评分 — 把 ROIC、再投资回报率、Net Debt/EBITDA、FCF Yield 打包为单值
  2. 资本配置历史追踪 — 历年 capex / 分红 / 回购 / 并购的"效率评分"
  3. 管理层"墓志铭"录入与对照 — 用户在研究时录一句话墓志铭，30 天后自动提醒对照
  4. "长期持有适宜度"标签 — 对每只标的计算 turnover 容忍度、年化综合摩擦
  5. 估值 vs 内在价值（DCF 简化版） — 用 owner earnings + 折现率 8-10% 倒推 fair value 区间

1.2 宏观驱动 / Macro-driven — Druckenmiller, Dalio, Soros, Kyle Bass

原型画像
先判断"宏观剧本"（汇率/利率/地缘），再决定"做多/做空什么"。仓位集中、决策快、止损极严。他们对单一公司研究很浅，但对链条的传导路径研究很深。

早晨先看什么

  • 全球 10 年期国债收益率 / 关键期限利差
  • 美元指数 + 关键货币对（USD/JPY, EUR/USD, USD/CNY）
  • 大宗商品（铜油金）的"形态"
  • 中央银行讲话日历 + 即将公布的宏观数据（CPI, NFP, PMI）
  • "市场叙事"（financial press 头条、KOL 的共识）— 目的不是信，而是找"共识到了哪一步"

筛选条件 / Anomalies

  • 政策套利：央行政策窗口期（加息末期、降息初期）的"夹层"机会
  • 财政赤字 / 国际收支双赤字国家货币的"死亡率"曲线
  • 大宗商品的"中国需求"替代信号（PMI, 房地产新开工, 铜油比）
  • 尾部风险定价（VIX term structure, OIS spreads, CDS 5Y 跳变）
  • 地缘事件后的"恐慌窗口" — 24-72 小时的流动性折价

深度工作流（拿到宏观剧本后的 2-5 步）

  1. 写"剧本"：用 3 段话写清"未来 6 个月我认为会发生什么" — 反向找 3 个反证
  2. 找"完美对冲" ：如果剧本错，错的方向怎么对冲
  3. 画"传导链"：央行政策 → 利率 → 汇率 → 出口/进口 → 行业利润 → 个股
  4. 算"不对称"：每个候选仓位的"上行/下行"赔率；只下注赔率 > 3:1
  5. 设"硬止损" — 60-90 天，最迟 90 天，平仓不论输赢

买入触发（pull the trigger）

  • 央行政策或财政事件让剧本"加速兑现"
  • 共识叙事的极端化（"这次不一样"）— 准备反向仓位
  • 流动性事件（基金赎回、ETF 流动性危机、carry trade unwind）

卖出信号

  • 剧本兑现 60% 以上
  • 硬止损触发
  • 出现"反向剧本"证据（央行转向、战争结束、技术性突破）
  • 出现"拥挤"信号（仓位调查、CFTC COT positioning）

对平台的 3-5 个可编码信号

  1. 宏观仪表板 — 利率、汇率、大宗、VIX、COT 持仓一屏可视化
  2. 政策日历 — 央行讲话、CPI/PMI/NFP 公布时间表 + 用户预判
  3. 剧本录入与对照 — 用户写 3 段剧本，60/90 天后自动推送回看
  4. 传导链可视化 — 选择"中国降息 25bp" → 自动列出受影响板块排序
  5. 赔率计算器 — 录入上下行情景与概率，自动算期望值 / 凯利仓位

1.3 事件驱动 / Catalyst — Ackman, Einhorn, Icahn, Bill Ackman 的 Pershing Square

原型画像
"找到即将发生且市场尚未充分定价的事件 — 然后等兑现"。Catalyst 可能是诉讼、监管、和解、分拆、并购、回购、CEO 变更、FDA 决定、宏观政策。

早晨先看什么

  • SEC EDGAR 8-K 当日列表
  • 法院文件 PACER（集体诉讼、判决、和解）
  • FDA Adcom 会议 / PDUFA 日期
  • 公司公告（中文语境：港交所披露易、深交所/上交所公告）
  • 法庭追踪（fdlive.com、RECAP 法律播客）
  • 管理层路演 / 投资者会议日程

筛选条件 / Anomalies

  • "Stub" 价值 vs Market cap — 分拆后被低估的母公司/子公司
  • 大量回购 + 管理层公开买入 + SEC 1% holder file 同向 — 强信号
  • 与现有持仓高度相关但折价 30%+ 的可转换债 / 优先股
  • 被做空 + 现金流稳定 + 短期 catalyst（盈利、回购、诉讼和解）
  • LBO / 分拆 前 6 个月 — 顾问/律师进场信号

深度工作流（拿到 catalyst 后的 2-5 步）

  1. 画时间轴 — catalyst 何时发生、概率多大、对应市值的隐含变动
  2. 算 IRR — "6 个月上涨 30% 下跌 10%，概率 60/40" → 年化多少？
  3. 找"同源催化" — 一个事件可能引发同行业 5 家公司重估
  4. 做"辩护" — 假设你看空，写出 5 个看空理由，看你能否反驳
  5. 设计 hedge — 如果是法律事件，用同业 / 行业 ETF 做 hedge；如果是 FDA 事件，用 basket trade 对冲

买入触发

  • Catalyst 时间确定 + 估值差距明显 + 下行有底
  • 多次 catalyst 叠加（盈利 + 诉讼和解 + 回购启动）
  • 卖空者集中度过高（> 20% SI / Days to cover > 10）

卖出信号

  • Catalyst 兑现 50-70%（不要等全兑现）
  • 出现"二次 catalyst"（诉讼延期、政策反复）但盘面已 price in
  • Better idea 出现（机会成本）

对平台的 3-5 个可编码信号

  1. 催化剂日历（Catalyst Calendar） — 8-K / 法院 / FDA / 股东大会 / 解禁日 全集
  2. "8-K 当日扫描" — 每天早上列出当日 8-K 公告，AI 摘要，标记相关持仓
  3. 集体诉讼与和解追踪 — 美股 PACER / A 股证监会处罚
  4. 回购 + insider buy + 财务杠杆三因子合表 — 一张表看资本信号集中度
  5. Stub 价值 / 分拆后低估检测 — 母公司 vs 子公司市值 vs 净资产对比

1.4 量化系统 / Systematic Quant — Jim Simons, D.E. Shaw, AQR, Two Sigma

原型画像
"没有任何人类判断，所有规则都被代码化"。他们不"找 idea"，他们找到 alpha 因子 → 验证统计显著性 → 上线 → 持续监测衰减。

早晨先看什么

  • 策略 PnL 曲线 + Sharpe / 换手 — 昨夜每个因子的贡献
  • IC（信息系数）历史衰减图 — 哪些因子在衰减
  • 回撤归因 — 哪些品种 / 哪些因子贡献了昨日回撤
  • 交易成本 / 滑点监测 — 策略是否被 front-run
  • 数据源健康度 — 数据错误会立刻击穿 PnL

筛选条件 / Anomalies

  • 横截面因子：动量、价值、质量、低波、分红、carry、size、growth
  • 时间序列因子：趋势、波动率、均值回归
  • 事件因子：盈利超预期、分析师上调、分红增长、回购
  • 另类数据：信用卡刷卡量、卫星图像、夜间灯光、网页爬虫
  • 市场微观结构：order flow imbalance、bid-ask 异常、ETF creation/redemption 流量

深度工作流（拿到 alpha 假设后的 2-5 步）

  1. 回测 — 至少 10 年样本，避免 survivorship bias
  2. 样本外测试 — paper trade 6 个月
  3. 因子检验 — 因子收益对各宏观状态是否稳健
  4. decay 监控 — 因子上线后第 1/3/6/12 个月的 IC 衰减
  5. 组合再平衡 — 风险预算 + 因子配权 + 行业/市场中性

买入触发

  • 因子 trigger 触发（无人工判断）

卖出信号

  • 因子 trigger 反向
  • 风控阈值触发（单日 VaR 突破、相关性突变）

对平台的 3-5 个可编码信号

  1. 因子库 — 价值/质量/动量/低波/分红/分析师上调 各打分
  2. 回测引擎 — 已有。需强化：survivorship-bias-free、多空、点-in-time
  3. 策略衰减监控 — 任何因子的 IC / IR 时序图，三个月下滑超阈值自动告警
  4. 组合风险归因 — 单日 PnL 按因子、品种、杠杆归因
  5. 市场微观结构信号 — 资金流、大单异动、北向资金、ETF 申赎

1.5 主题 / 长期趋势 — Cathie Wood, Chamath Palihapitiya, ARK 团队

原型画像
"在 5-10 年尺度上找颠覆性技术 / 商业范式迁移"。他们的研究对象是生态系统（"创新平台"），不是单只股票 — 平台上的所有公司都可能是持仓。

早晨先看什么

  • 专利数据库（USPTO / WIPO）— 看技术趋势
  • 学术论文 / arXiv / 顶会 — 跟踪研究前沿
  • 行业研究报告（Gartner, McKinsey, a16z）
  • 早期融资数据库（Pitchbook, Crunchbase）— 看资金流向
  • 政府合同 / NASA / DARPA / 中国十四五规划 — 政策导向

筛选条件 / Anomalies

  • "S-curve" 早期 — 渗透率 < 5% 但加速度 > 0
  • TAM/SAM 5 年 10x 的市场
  • "平台公司" — 拥有开发者 / 卖家 / 内容生态
  • 关键瓶颈 被解除（如电池成本、芯片良率、监管放开）
  • 管理层有技术背景 + 长期持股

深度工作流（拿到主题后的 2-5 步）

  1. 画"价值链" — 上游材料 → 中游设备 → 下游应用
  2. 找"瓶颈节点" — 整个链里最稀缺 / 最卡脖子的环节
  3. 找"利益相关者" — 政策制定者、监管、行业协会
  4. 跟踪关键 milestone — 技术路线图、产品发布、监管节点
  5. 设 5 年时间窗口 — 期间允许主题内个股频繁换手

买入触发

  • 渗透率拐点 + 政策催化
  • 关键技术突破 / 量产信号
  • 大客户签约 / 国家政策落地

卖出信号

  • 渗透率到 30%+，拥挤交易出现
  • 出现更优替代技术
  • 公司失去生态位置

对平台的 3-5 个可编码信号

  1. 主题/产业链图谱 — 节点关系图，每个节点挂相关公司
  2. 政策追踪 — 监管 / 补贴 / 五年规划关键词命中
  3. 专利 / 论文热度趋势 — 关键词在 arXiv / USPTO / 国家知识产权局 的频次
  4. 渗透率估算 — 主题相关指标（如电动车销量、机器人装机量）
  5. 早期融资信号 — Pitchbook / IT 桔子 / Crunchbase 抓取

1.6 特殊情境 / Special situations — Greenblatt, Einhorn 早期, Joel Greenblatt 式的"魔法公式", Spin-off 套利

原型画像
"找 3-18 个月内必然发生的事件，市场因'技术性折价'给的机会"。典型场景：

  • 分拆（Spin-off） — 母公司跌 5%，分拆公司上市 30 天内有 5-15% 折价
  • 破产重组（Distressed） — 8 债转债 / 银行股 / 困境反转
  • 并购套利（Merger Arb） — 公告后买入目标公司，扣减 spread
  • 现金返还特别大 — 大股东大额减持 / 一次性大额分红

早晨先看什么

  • Spin-off 公告（10-K, 新闻） — 每季度 < 5 个，但每个都是机会
  • 破产法院文件 — 重组进度 / 债权人委员会
  • 并购公告 + 监管审批节点（HSR, 欧盟, FTC）
  • 公司债 / CDS 利差
  • SPAC / 信托账户价值

筛选条件 / Anomalies

  • 公告后的 spread > 8% （merger arb）
  • 分拆后 30 天内 trading discount > 8%
  • 公司债 yield > 15% + 重组方案明确
  • 现金 / 市值 > 50% 的"现金壳"
  • 大股东协议转让折价 > 20%

深度工作流（拿到特殊情境后的 2-5 步）

  1. 看退出条款 — deal break fee、融资确定性、监管
  2. 算"时间价值" — 假设 6 个月完成，IRR 多少
  3. 找"同源" — 同一 deal 的可转债 / 优先股 / 期权
  4. 算"风险价值" — deal 失败的损失
  5. 用 options 替代 stock — 锁定下行

买入触发

  • 折价充分 + 退出明确
  • 监管节点前 1-2 个月

卖出信号

  • 退出完成
  • deal 失败

对平台的 3-5 个可编码信号

  1. 分拆追踪表 — 母公司代码、分拆日、分拆后 trading discount 时序
  2. 并购套利计算器 — 公告价、当前价、退出时间、deal break fee → IRR
  3. 破产重组进度 — 8 章 / 11 章 关键节点
  4. 大额协议转让 / 大宗交易追踪 — 折价率
  5. 现金 / 市值比扫描 — 整个 A 股 / 美股 一键筛"现金壳"

1.7 A 股本土实务 — 民间股神、私募操盘手、产业链调研派

原型画像
A 股是一个信息结构极度不对称、政策驱动、散户主导（> 60% 交易量）的市场。优秀的本土投资者极少单纯看财报，而是"产业链 + 政策 + 资金 + 情绪"四因子同时建模。

早晨先看什么

  • 政策信号：央行 / 财政部 / 行业部委 / 地方政府的"会议通稿"与"工作部署" — 关键看哪些词首次出现 / 消失
  • 交易所互动易 — 上市公司回答投资者问题（很多是官方套话，但部分提前透风）
  • 券商研究所晨会纪要 — 大型券商如中信、华泰、中金的"晨会"与"产业链调研纪要"
  • 资金面：北向资金、融资融券余额变化、ETF 申赎、龙虎榜
  • 草根调研 — 上市公司公开的"投资者关系活动记录表"

筛选条件 / Anomalies

  • "政策预期" — 政治局会议 / 中央经济工作会议 / 行业大会前 1-2 周，对应板块的"提前定价"
  • "国家队"动向 — 证金 / 汇金 / 国新投资 的 ETF 持仓变化、央企 / 国资委旗下公司的增减持
  • 产业资本 — 大股东回购、董监高增持、员工持股计划
  • "妖股记忆" — 历史上相似题材（O2O、5G、ChatGPT、低空经济、固态电池）的炒作时间表
  • "打板" 后的二板定龙头 — 涨停后回撤 30%+ 的"龙头"经常有第二波
  • "高股息 + 国资" 套利 — 红利低波 ETF 拥挤后的反向

深度工作流（拿到一个"题材"后的 2-5 步）

  1. 政策回测：过去 5 年类似政策的板块反应幅度
  2. 资金面 ：北向 + 融资 + 主力净流入 三者共振
  3. 产业链：上游 / 中游 / 下游 的弹性排序
  4. "龙头/跟风"识别：先涨 30% 但回撤 < 15% 的是龙头
  5. 退出计划 — 政策落地后 3-5 天无论盈亏清仓（"利好出尽"）

买入触发

  • 政策首次提及 + 资金开始净流入 + 龙头股已涨停
  • 跌破定增价 / 大股东增持价 / 股权激励行权价
  • 大股东自愿延长锁定期

卖出信号

  • 政策落地 = 利好兑现
  • 龙头滞涨、跟风股补涨 = 板块见顶
  • 监管降温（特停、问询函、二级市场提示风险）
  • 出现"高位巨量阴线"

对平台的 3-5 个可编码信号

  1. 政策关键词命中 + 历史回测 — 关键词首次出现 → 列出过去 5 年类似表述的板块反应
  2. 资金面三因子合表 — 北向 + 融资 + 主力 净流入共振
  3. "国家队" / 国资委 / 央企 持仓变化追踪
  4. 董监高 / 大股东增减持 + 解禁 + 回购 一张表
  5. 定增 / 股权激励 / 员工持股 折价率扫描

1.8 加密 / 数字资产 — 这一块可作为"附加原型"

原型画像

  • 链上数据派：看交易所净流出、巨鲸地址、stablecoin 发行量
  • 衍生品派：资金费率、基差、隐含波动率
  • VC / 周期派：每一轮周期（BTC 减半、ETH 升级）的"前后节奏"

对平台的 3-5 个可编码信号

  1. 交易所 BTC/ETH 净流出 — 30 日累计
  2. 巨鲸地址变动 + 稳定币市值
  3. 资金费率 / 持仓量
  4. DApp 活跃地址 / TVL
  5. 周期指标 — MVRV、AHR99、Puell Multiple

2. 平台功能地图（Section A）

把上文的"方法论"翻译成"可点击的 UI"。每个功能给出：原型归属、UI 形态、数据依赖、复杂度（S/M/L）、用户价值（1-5 星）。

F01. 晨间一页（Morning Brief）

  • 原型归属：所有
  • UI 形态：Dashboard 顶部固定卡片。每天 8:30 推送 / 生成：
    • 隔夜美股 / 港股 / 大宗 / 汇率
    • 持仓相关公告（8-K / 港交所披露易 / 互动易）
    • 即将到来的催化剂（5 个最近日期）
    • 政策与宏观日历（今日 / 明日）
    • AI 摘要 1 段（平台 LLM，默认 MiniMax / DeepSeek legacy）：昨晚市场叙事 + 今日关注
  • 数据依赖：宏观（FRED/yfinance）+ 公告（EDGAR/港交所）+ 平台 scheduler
  • 复杂度：M
  • 价值：★★★★★

F02. 异常扫描器（Anomaly Scanner）

  • 原型归属：复合价值、量化、本土实务
  • UI 形态：独立页面 + 每日推送。多 tag 过滤：
    • 量价异常：5 日均量 ×3，价量背离
    • 北向 + 融资 + 主力 三共振
    • 涨停 / 跌停 + 行业集中度
    • 估值分位 < 5% 或 > 95%
    • 高管 / 大股东 集中增持（5 人 30 天内）
  • 数据依赖：行情 + 资金面 + 公告（已有 akshare/yfinance，需补 EDGAR/港交所）
  • 复杂度：L
  • 价值：★★★★★

F03. 催化剂日历（Catalyst Calendar）

  • 原型归属：事件驱动、特殊情境
  • UI 形态：时间轴 + 列表 + Tag
    • 盈利、FDA、股东大会、分拆、并购审批、8-K、法院、和解、解禁、分红
    • 每条可订阅，催化前 24h / 7d 自动推送
  • 数据依赖：SEC EDGAR 8-K（已部分有，需补）/ 港交所披露易 / 互动易 / FDA Adcom
  • 复杂度：L
  • 价值：★★★★★

F04. 13F Diff Tracker（机构持仓变动）

  • 原型归属：量化、事件驱动
  • UI 形态：季度 diff 视图
    • 标的形式：XX 机构 Q2 持仓变化（新增 / 加仓 / 减仓 / 清仓）
    • "Whale Cluster"：3+ 头部基金同向加仓
    • "Consensus Move"：> 10 家基金同向
  • 数据依赖：13F（whale wisdom / 13D / 13G）
  • 复杂度：L（数据获取是最大瓶颈）
  • 价值：★★★★★

F05. 暗池 / 大宗交易监测

  • 原型归属：量化、本土实务
  • UI 形态：单页
    • 美股：暗池 block trade、options unusual activity
    • A 股：大宗交易折价榜、ETF 申赎、龙虎榜
  • 数据依赖：Unusual Whales / CBOE / akshare
  • 复杂度：L
  • 价值：★★★★

F06. 内部人集中买入检测

  • 原型归属：复合价值、事件驱动
  • UI 形态：filter chip
    • 3 名以上董事/高管 30 日内增持
    • 增持金额 > 市值 0.1% 或 > 1000 万
    • Form 4 / 港交所权益披露
  • 数据依赖：Form 4 / 港交所权益披露（已有 akshare 部分）
  • 复杂度：M
  • 价值：★★★★

F07. 做空激增 / SI Spike

  • 原型归属：事件驱动
  • UI 形态：filter chip
    • SI > 20% 且 30 日内上升 50%
    • Days to cover > 10
    • Borrow fee > 5%
  • 数据依赖：FINRA / Ortex
  • 复杂度：M
  • 价值：★★★★

F08. 盈利超预期历史

  • 原型归属：事件驱动、复合价值
  • UI 形态：详情页新增
    • 过去 8 季 beat/miss 历史
    • 公布后 1d / 5d / 30d 收益分布
    • "Surprise Magnitude" vs 反应幅度
  • 数据依赖：FactSet / Zacks / 历史股价
  • 复杂度：M
  • 价值：★★★★

F09. 预期修订追踪（Estimate Revision）

  • 原型归属：量化、事件驱动
  • UI 形态：filter chip
    • 30 日内 4+ 分析师上调 EPS
    • 上调 / 下调 比 > 2
    • EPS 修订幅度 top 5%
  • 数据依赖：FactSet / Zacks / IBES
  • 复杂度：M
  • 价值：★★★★

F10. 杠杆 / 财务压力异常

  • 原型归属：复合价值、特殊情境
  • UI 形态：filter chip
    • 净负债 / EBITDA > 4x
    • Z-Score < 1.8
    • Altman / Beneish M-Score
    • 利息保障倍数 < 2
  • 数据依赖：财报数据
  • 复杂度：S（数据已在，逻辑写）
  • 价值：★★★

F11. 板块轮动热力图（Sector Rotation Heatmap）

  • 原型归属：宏观、本土实务
  • UI 形态：可视化
    • 横轴：行业 / 主题
    • 纵轴：1w / 1m / 3m / 6m / 1y 相对强弱
    • 颜色：跑赢 / 跑输 S&P
    • 标记：RS 新高 / 新低
  • 数据依赖：行情
  • 复杂度：S
  • 价值：★★★★★（已有部分）

F12. 新闻情绪 vs 价格背离

  • 原型归属：量化、宏观
  • UI 形态：双轴图
    • 上方：价格
    • 下方：新闻情绪指数（AI 打分）
    • 标记：情绪极值
    • 背离信号：情绪高 + 价格跌 = 抄底？情绪低 + 价格涨 = 出货？
  • 数据依赖：新闻 + AI 摘要（已有 LLM 集成：新闻情绪管线用 DeepSeekProvider，平台默认 MiniMax）
  • 复杂度：M
  • 价值：★★★★

F13. 管理层质量看板（Capital Allocation Scorecard）

  • 原型归属：复合价值
  • UI 形态：详情页新增
    • 历年 capex / 分红 / 回购 / 并购的 ROI
    • 高管持股比例 + 历史变动
    • 关联交易频繁度
    • 股权激励 vs 业绩匹配

  • 数据依赖：财报 + 高管履历 + 关联公告

F14. 客户 / 收入集中度

    • 节点：公司 / 行业 / 主题

4. 工作流模板（Section C）

每个原型的"如果我是 XX 投资者，今天早上 8:30 我点哪里"。这些将作为应用内快速入门。

4.1 复合价值 / Terry Smith 式投资者

08:30  打开 Morning Brief → 关注 "持仓相关公告" + "政策日历"
09:00  进入"持仓" → 每只标的开"墓志铭"标签
09:30  Anomaly Scanner → 过滤 "高质量 + 合理价"（ROIC > 20% + FCF Yield > 5% + Net Debt/EBITDA < 2x）
10:00  对命中标的：录入"剧本"（30 / 60 / 90 天回看）
10:30  Catalyst Calendar → 看未来 90 天是否有 "管理层增持 / 回购 / 分红" 类催化
11:00  Capital Allocation Scorecard → 计算"再投资回报率"是否仍 > WACC
13:00  收盘后：检查已有剧本是否有 30 天到期 → 触发回看

4.2 宏观驱动 / Druckenmiller 式投资者

08:00  Macro Dashboard → 看 US 10Y、DXY、CU/AU 比、VIX
08:15  Catalyst Calendar → 标出"今日央行讲话 / 重要数据"
08:30  Morning Brief → 读 AI 摘要
09:30  Anomaly Scanner → 用"宏观标签"（如 DXY 突破、US 10Y 突破）筛 A 股 / 美股
10:00  传导链图 → "中国降息 25bp" 模拟 → 列出受益板块
10:30  录入"剧本"：3 段话 + 反证 + 关键节点
11:00  Kelly 仓位工具：计算"政策兑现 vs 失败"赔率
13:00  收盘后：检查剧本是否触发 / 反证是否出现

4.3 事件驱动 / Ackman 式投资者

08:00  Catalyst Calendar → 按"日期"排序最近 90 天
08:30  8-K 当日扫描 → 标记相关持仓
09:00  Anomaly Scanner → "内部人 3+ 增持 + 大量回购 + SI > 15%"
09:30  分拆追踪表 → 查 "30 日内分拆" 的 trading discount
10:00  并购套利计算器 → 列出 spread > 8% 的 deal
10:30  13F Diff Tracker（Phase 3）→ 看 3+ 头部基金同向加仓
11:00  对命中候选：录入剧本 + 60 天回看
13:00  收盘后：检查催化前 24h 候选清单

4.4 量化 / Simons 式投资者

08:00  打开"策略 PnL 面板" → 看昨夜每个因子的 IC
08:15  因子库 → 检查每个因子的 30/60/90 日 decay
08:30  Anomaly Scanner → 资金面 + 微观结构类信号
09:00  行业轮动 → 横截面因子相对强弱
09:30  对新因子：单因子回测 → paper trade 6 个月
10:00  组合归因 → 单日 PnL 按因子 / 品种归因
10:30  Risk Dashboard → VaR / 杠杆 / 相关性矩阵
13:00  收盘后：组合再平衡 / 风险预算

4.5 主题 / Cathie Wood 式投资者

08:00  政策追踪 → 关注"首次出现的关键词"
08:30  主题图谱 → 选择主题（如"低空经济"）→ 列出节点
09:00  Anomaly Scanner → "渗透率 < 5% + 政策催化"
09:30  催化日历 → "FDA / 产品发布 / 大客户签约"
10:00  主题深度报告（AI）→ 平台 LLM（默认 MiniMax）生成

5. 关键的非显而易见的洞见

写到这里已经把"信号 → 功能 → 工作流"都覆盖了。但还有几件事值得单独说。

5.1 "数据获取"是真正的瓶颈，不是"计算"

F01-F20 中，80% 的"难度"来自数据源，而非计算或 UI。F04（13F）、F05（暗池）、F07（SI）、F09（分析师预期）都需要付费 API 或专业爬虫。建议：

  • Phase 1 集中做"已能计算"的功能（F11、F17、F18、F10、F14、F12）
  • Phase 2 / 3 把 30% 的工程投入放在"数据 ETL + 缓存"，而不是"功能 UI"
  • 2026 下半年至少 1-2 个付费数据源合作（如 Whale Wisdom、Finnhub Premium、Unusual Whales）

5.2 "剧本录入"比"AI 研报"更值得做

LLM 让"AI 研报"看似很美，但 90% 的研报对您来说是噪音。您已经能识文断字、能写分析，真正稀缺的是"把您自己的判断结构化"。F17（剧本录入 + 自动回看）是投入产出比最高的功能 — 实施成本 S，价值 5 星，且训练您自己的判断纪律。

5.3 "工作流模板"是留存关键

当一个用户能"按一个按钮、按 10 步完成我今天的研究" — 他很难离开这个平台。

把 Section 4 落到应用内是提升 DAU / 留存最便宜的方式。每个原型的"10 步"可以做成侧边栏固定入口，或多端同步的 PWA 入口。

5.4 "组合"才是核心，"标的"是次要的

Soros / Druckenmiller 反复强调：重要的不是你对了多少次，而是错的时候亏多少。所以：

  • 风险归因 / VaR / 凯利仓位类功能（F18）应该比"找新 idea"功能更靠前
  • Phase 1 必须有 "组合压力测试" — 利率 + 100bp、汇率 - 5%、GDP - 2% 的组合 PnL
  • Phase 2 必须有 "最大回撤 vs 同质替代" — 您的组合在历史上的 5 个熊市回撤分别是多少

5.5 "时间衰减"是您最该监控的元指标

每加一个功能，请记录：

  • 该功能上线后第 1 / 3 / 6 个月的使用率
  • "用户对 X 信号的采纳率"（看到 → 录入剧本 → 60 天后回看 vs 没回看）

如果用户不看回看，F17 就失败了。永远不更新比"3 个月没新功能"更糟糕。

5.6 跨原型共有的"金发姑娘区"

观察 5 个原型的买入触发，您会发现它们都指向"赔率 + 催化 + 不对称"这个三元组：

原型 赔率 催化 不对称
复合价值5-10 年复合估值杀 + 基本面不变下行有底
宏观3-5 倍上行央行/政策60 天硬止损
事件驱动30-50% 上行8-K / FDAdeal break risk
主题5-10x 长期渗透率拐点主题内换手
本土实务1-3 倍短线政策首次提及政策兑现后清仓

这意味着：F18（赔率计算器）几乎对所有原型都有价值。F03（催化剂日历）也几乎是全原型通用。这两个应该是 Phase 1 必上。

5.7 平台不该做、但用户必须自己做

  • "判断" 本身 — 平台能给您所有信号，但"信号 A 与信号 B 矛盾时该怎么办"只有您能决定
  • "耐心" — 平台不能帮您忍住不在 30 天冷静期买
  • "逆共识的勇气" — 平台能列"市场共识"与"反方观点"，但下单按钮永远在您手上
  • "退出" — 卖出的难度是买入的 5 倍，平台能提示，但执行靠您
