## 1. 平台概览

### 1.1 一句话定位

AD-Research 是一个**自部署、多市场、智能体驱动的投研与回测一体化平台**：把 A 股 / 港股 / 美股 / 加密货币 / 期货的行情、微结构、宏观、新闻、社交舆情聚合到一个 Web 控制台，配合本地策略引擎和 AI 研报生成，让用户能够完成"选股 → 验证 → 监控 → 自动化执行"的完整闭环。

### 1.2 核心价值

| 维度 | 价值主张 |
|---|---|
| **多市场覆盖** | A 股、ETF、美股、加密、商品期货、宏观指标一站式拉通（不依赖单一数据商） |
| **复权/行情一致** | 统一 raw close + adj_factor 双轨存储，前端按需展示复权 |
| **可观测 ETL** | 每个 pipeline 写入 `etl_logs` 表、`result.success/records`、Redis 锁保护互斥 |
| **AI 增强** | MiniMax（默认）用于研报摘要、雪球内容深度抓取、聊天助手；DeepSeek（legacy）与 Anthropic Claude 备用 |
| **回测 / 实盘** | 支持基于历史数据的回测与基于信号的模拟仓（paper trading） |
| **单点登录 + 多设备** | JWT access token + refresh token + user_device 多端管理 |

### 1.3 用户群

- **主要**：个人投资者 / 量化研究员，需要全市场扫描 + 自定义策略
- **次要**：私募 / 投顾团队的桌面看板（数据本地掌控，便于合规）
- **内部**：用于 debug 数据源问题、规划新数据源的"运维视角"

### 1.4 技术栈速览

| 层 | 技术 |
|---|---|
| 后端 | Python 3.11 + FastAPI + SQLAlchemy 2.x + APScheduler 3.x |
| 数据库 | PostgreSQL 16（数据）+ Redis 7（缓存 + 分布式锁 + Celery broker） |
| 前端 | React 18 + TypeScript + Vite + Ant Design v5 |
| 状态/数据 | TanStack Query（React Query）+ Zustand + Ant Design 主题 |
| 部署 | Docker Compose + GitHub Actions self-hosted runner + Nginx |
| LLM | MiniMax（默认，OpenAI 兼容）+ DeepSeek（legacy）/ Anthropic（备），统一 `app/services/llm/` |
| 抓取 | requests + BeautifulSoup + Playwright（雪球 Cookie 路由） + RSS |

## 6. 关键技术决策

### 6.1 复权：raw close + adj_factor 双轨

- `daily_bars.close` **永远存原始收盘价**，不复权。
- 单独列 `adj_factor` 由 Tushare/akshare 提供。
- 前端查行情时，后端按 `adj=qfq/hfq/none` 即时计算。
- **好处**：避免历史回测数据被改写、所有指标可重算、跨数据源一致性。

### 6.2 Pipeline × Provider × PipelineResult 三层架构

```
Provider（外部数据获取）
   ↓  (raw dict / df)
Pipeline（清洗 + 去重 + upsert）
   ↓  PipelineResult(success, records, errors)
Scheduler 调度 + redis_lock 互斥 + etl_logs 入库
```

每个 pipeline 继承 `BasePipeline.run_with_retry`，带 1-3 次自动重试 + 异常吞咽（部分域）。

### 6.3 Agent worker 架构（雪球）

`scripts/xueqiu_playwright_worker.py` 是一个**独立进程的 Playwright worker**，监听 Redis 队列，模拟浏览器登录态抓取需要 Cookie 的雪球内容。
- 主进程 API 端点 `POST /api/v1/news/xueqiu/refresh` 把任务投递 Redis
- worker 进程消费、爬取、把结果写 DB
- **为什么独立**：
  1. Cookie 失效可单独重启 worker，不影响 web
  2. 浏览器有内存泄漏风险，独立进程便于 `kill -9`
  3. 多节点部署可水平扩展

### 6.4 LLM 抽象层

`app/services/llm/`，所有 LLM 调用通过统一接口：

```python
class LLMService:
    chat(messages, stream=False) → str | AsyncIterator
    summarize(text, max_tokens=200) → str
```

实现 `minimax_provider.py`（默认）、`deepseek_provider.py`（legacy）与 `anthropic_provider.py`，由 `LLM_PROVIDER` 配置切换。

### 6.5 数据库迁移

- **Alembic** 作主迁移工具，迁移文件 `alembic/versions/`
- 每个 phase 自带 migration，回滚顺序记录在 `docs/dev-notes/`
- 部署时容器启动自动 `alembic upgrade head`

### 6.6 行锁 + 唯一约束去重

- ETL 写入用 `(code, trade_date)`、`(announcement_id)` 等 unique 约束确保幂等
- 长任务用 `app/core/redis_client.py:redis_lock(name, expire)` 互斥

### 6.7 时区统一

- **DB 全部 Asia/Shanghai**（`TIMESTAMP WITH TIME ZONE` 不允许；用 `DATETIME` 存裸北京时间）
- 序列化时统一 ISO-8601 + `+08:00`
- 前端用 dayjs，所有 `dayjs.tz('Asia/Shanghai')`

## 附录 B：角色 / 权限矩阵

| 角色 | 权限 |
|---|---|
| user | 全功能（默认角色，`users.role` 默认值） |
| admin | + `/admin/users` + `/admin/deployments` + 真实交易、ETL 运维看板等 admin-only 端点 |

> 代码中只有 `user` / `admin` 两种角色（`app/api/deps.py` 以 `role == "admin"` 判定）；未登录请求一律被 JWT 拦截，无 guest 角色。
> JWT access token 过期默认 1440 分钟（24h，`ACCESS_TOKEN_EXPIRE_MINUTES = 60*24`），refresh token 30 天。

## 附录 C：术语表

| 术语 | 含义 |
|---|---|
| **raw close** | 未经复权处理的当日实际收盘价 |
| **adj_factor** | 复权因子；qfq = 前复权 / hfq = 后复权 |
| **pipeline** | 数据 ETL 任务单元，自带 retry + lock |
| **provider** | 单一数据源的客户端封装 |
| **pool** | 用户自定义标的集合（规则或手动） |
| **strategy** | 预定义的信号生成算法 |
| **signal** | 由 strategy 在某个 (策略,标的,日期) 触发的买卖提示 |
| **paper trade** | 不下单到真实券商、仅在本地模拟成交 |
| **agent worker** | 独立进程的特殊数据采集器（雪球/Playwright） |
