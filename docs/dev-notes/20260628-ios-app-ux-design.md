# iOS APP UI/UX 原型设计文档

> 2026-06-28 | 基于投研平台功能，原生 Swift 实现

---

## 一、设计原则

### 1.1 核心原则

| 原则 | 说明 |
|---|---|
| **本地优先** | 打开即有数据，网络请求不阻塞 UI |
| **手势优先** | 用原生手势替代 Web 端按钮操作 |
| **信息分层** | 重要信息前置，详情按需展开 |
| **触控友好** | 44pt 最小热区，不留死角 |
| **统一主题** | 与 Web 端 Swiss Minimal + Neon Cyan 对齐 |

### 1.2 与 Web 端的关键差异

| 维度 | Web 端 | iOS App |
|---|---|---|
| 导航 | 固定侧边栏 + 页面跳转 | 底部 TabBar + NavigationStack push |
| 信息密度 | 大屏，表格为主 | 小屏，卡片 + 列表为主 |
| 交互 | 点击、hover、右键菜单 | 点击、长按、滑动、捏合 |
| 操作 | 多按钮并列 | 关键操作突出，次要收进更多菜单 |
| 刷新 | 页面加载 | 下拉刷新 / 自动刷新 |
| 状态提示 | 悬浮 tooltip | Haptic 震动 + Toast |

### 2.2 交互模式

| 页面类型 | 交互模式 |
|---|---|
| 列表页 | 下拉刷新 + 上拉加载更多 + 长按快捷操作 |

## 四、iOS 原生交互规范

### 4.1 导航手势

| 手势 | 场景 | 反馈 |
|---|---|---|
| 右滑返回 | 所有 push 页面 | 跟随手指动画 |
| 下拉 | 列表刷新 | RefreshControl 旋转 |
| 上拉 | 加载更多 | 底部 Loading 指示器 |
| 捏合 | K线缩放 | 实时缩放 |
| 长按 | 列表项/图表 | Context Menu + Haptic |
| 左滑 | 列表项 | 操作按钮（删除/收藏） |
| 双指 | K线区域选择 | 选中放大 |

### 4.2 Haptic 触觉反馈

| 场景 | 反馈类型 |
|---|---|
| 登录成功 | `.notification(.success)` |
| 收藏/取消收藏 | `.impact(.light)` |
| 下拉刷新完成 | `.impact(.medium)` |
| 删除确认 | `.notification(.warning)` |
| 价格预警触发 | `.notification(.error)` |
| 开关切换 | `.selection` |

### 4.3 加载状态

| 状态 | 展示 |
|---|---|
| 首次加载 | 骨架屏（Skeleton — 灰色闪烁块） |
| 刷新 | 下拉 RefreshControl |
| 局部更新 | 半透明 overlay + 小 spinner |
| 网络错误 | 内联错误卡片 + 重试按钮 |
| 空数据 | 居中插画 + 引导文字 |
| 离线 | 顶部横幅 "离线模式 · 数据可能有延迟" |

### 4.4 字体系统

| 用途 | 字体 | iOS 实现 |
|---|---|---|
| 大标题 | 26pt Medium | `.largeTitle` |
| 页面标题 | 20pt Medium | `.title` |
| 段落标题 | 16pt Medium | `.title3` |
| 正文 | 13pt Regular | `.body` |
| 辅助文字 | 11pt Regular | `.caption` |
| 价格数字 | 24pt Mono | 自定义 `.system(size: 24, design: .monospaced)` |
| K线标签 | 10pt Mono | 自定义 |

**与 Web 端 theme.css 对齐：**
```

```
