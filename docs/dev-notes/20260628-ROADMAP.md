## 一、项目概述

### 1.1 目标
将手工化的 ETF 投研工作流升级为**一体化 Web 投研平台**，支持全市场 ETF 的自动化跟踪分析、标的池管理、技术指标计算、策略回测和信号生成。

### 1.2 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│  子项目4：策略系统 ✅ 已完成                                       │
│  ── 策略配置、回测引擎、信号生成、绩效归因                         │
├─────────────────────────────────────────────────────────────────┤
│  子项目3：Web 仪表盘 ✅ 已完成                                     │
│  ── 前端界面、图表可视化、交互式分析                               │
├─────────────────────────────────────────────────────────────────┤
│  子项目2：ETF 投研核心 ✅ 已完成                                   │
│  ── 全市场扫描、标的池增强、筛选排名、报告生成、综合评分             │
├─────────────────────────────────────────────────────────────────┤
│  子项目1：数据基础设施 ✅ 已完成                                   │
│  ── 数据采集管道、PostgreSQL 存储、ETL、指标计算、REST API          │
└─────────────────────────────────────────────────────────────────┘
```

### 3.1 功能优先级划分

基于**依赖关系**和**用户价值**，将 7 个功能分为 **P0（核心）** 和 **P1（进阶）**：

#### P0 核心功能（子项目2 实现）

这 4 个功能形成**"报告生产流水线"**：评分打分 → 筛选排序 → 池分析 → 生成报告

| 序号 | 功能 | 说明 | 依赖 |
|------|------|------|------|
| 1 | **综合评分系统** | 基于收益、风险、夏普、流动性等多维度给 ETF 打分 | 子项目1 的指标数据 |
| 2 | **全市场筛选排名** | 按评分/指标多条件筛选，生成 Top N 排名 | 评分系统 |
| 3 | **标的池增强** | 权重配置、持仓分析、池内对比、池间对比 | 子项目1 的池基础功能 |
| 4 | **报告生成引擎** | 定期自动生成 HTML/Markdown 研究报告，替代手工报告 | 评分 + 筛选 + 池分析 |

#### P1 进阶功能（子项目3 实现）

| 序号 | 功能 | 说明 | 依赖 |
|------|------|------|------|
| 5 | **板块/行业轮动分析** | 跟踪板块相对强弱、动量轮动信号 | 大量历史数据计算 |
| 6 | **全市场自动扫描** | 定期对比 akshare 列表和 DB，自动发现新增/退市/变更 ETF | 相对独立，可插入 ETL |
| 7 | **报告定时推送** | 邮件/企业微信等渠道自动推送生成的报告 | 报告引擎就位后 |

### 3.2 P0 功能详细说明

#### 3.2.1 综合评分系统

**目标：** 为每只 ETF 计算一个综合得分，便于横向对比。

**评分维度（建议）：**

| 维度 | 权重 | 指标 | 说明 |
|------|------|------|------|
| 收益能力 | 30% | return_1m, return_3m, return_1y | 近期和长期收益 |
| 风险控制 | 25% | volatility_20d, max_drawdown_1y | 波动和回撤 |
| 风险调整收益 | 25% | sharpe_1y | 夏普比率 |
| 流动性 | 10% | 日均成交额 | 避免流动性陷阱 |
| 趋势强度 | 10% | rsi14, ma5/ma20 位置 | 技术趋势 |

**输出：** 每只 ETF 的综合得分（0-100）和分项得分。

**存储（规划）：** ~~`etf_indicators` 表新增 `composite_score`, `score_return`, `score_risk`, `score_sharpe`, `score_liquidity`, `score_trend` 字段。~~
**实际落地：** 独立 `scores` 表（`app/models/scoring.py`，含 `composite_score` /
`score_return` 等分项列），按 (code, date) 维度落库。

#### 3.2.2 全市场筛选排名

**目标：** 基于多条件筛选 ETF，生成各类排名榜单。

**筛选条件：**
- 市场（A股/港股/美股）
- 分类（科技/周期/消费等）
- 指标范围（rsi_min/max, sharpe_min, volatility_max 等）
- 评分门槛（score_min）

**排名类型：**
- 综合评分 Top N
- 收益排名（1月/3月/1年）
- 风险最低排名（波动率/回撤）
- 夏普比率排名
- 趋势最强排名（RSI）

**输出：** JSON API + 嵌入报告。

#### 3.2.3 标的池增强

**目标：** 在子项目1 基础池功能上，增加投研所需的分析功能。

**新增功能：**

| 功能 | 说明 |
|------|------|
| **权重配置** | 池内每只 ETF 可配置目标权重（如 20%, 15%），实际权重偏离时提示 |
| **持仓分析** | 池的行业/板块分布、市值分布、QDII 占比等 |
| **池内对比** | 池内各 ETF 的收益/风险/相关性对比矩阵 |
| **池间对比** | 多个池的综合表现对比（收益曲线、回撤对比） |
| **再平衡提示** | 当实际权重偏离目标权重超过阈值时，提示需要再平衡 |

**数据模型变更：**
- `pool_members` 表新增 `target_weight`（目标权重）
- 新增 `pool_snapshots` 表（定期记录池的持仓快照，用于历史对比）

#### 3.2.4 报告生成引擎

**目标：** 替代用户目前手工生成的 HTML 报告，实现全自动定期生成。

**报告类型：**

| 报告 | 频率 | 内容 |
|------|------|------|
| **全市场扫描报告** | 每周 | 全市场 ETF 排名、新上榜/落榜、异常波动 |
| **标的池分析报告** | 每周 | 池内持仓分析、收益对比、风险提示 |
| **综合日报** | 每日 | 关键指标快照、持仓变化、市场概览 |

**报告内容模块（以周报告为例）：**

1. **市场概览** — 市场整体涨跌、成交额、情绪指标
2. **全市场排名** — 综合评分 Top 20、各行业最佳
3. **标的池分析** — 池内持仓表现、权重偏离、相关性矩阵
4. **重点关注** — 评分变化大的 ETF、新上榜/落榜、异常信号
5. **风险提示** — 高波动 ETF、低流动性警告、相关性过高的组合

**输出格式：**
- HTML（可浏览器直接打开，类似现有手工报告）
- Markdown（便于存档和版本管理）
- JSON（供前端仪表盘消费）

**报告存储：** 文件系统（reports/ 目录）+ 可选的 `reports` 表（记录报告元数据）

**定时触发：** 通过 APScheduler 在每周日晚上自动生成。

## 六、P1 进阶功能详细说明（供日后参考）

### 6.1 板块/行业轮动分析

**目标：** 识别当前市场热点板块和轮动趋势。

**分析方法：**
- 按 category 分组计算板块收益
- 计算板块相对强弱（RS）
- 识别动量领先的板块（过去 N 周收益排名）
- 板块轮动信号（领先板块切换时提示）

**输出：** 板块排名、轮动信号、热力图数据。

### 6.2 全市场自动扫描

**目标：** 自动发现新上市/退市/变更的 ETF，保持数据库与 akshare 同步。

**扫描逻辑：**
1. 从 akshare 获取最新 ETF 列表
2. 与 `etf_info` 表对比
3. 新增：插入新记录，status="active"
4. 退市：更新 status="delisted"
5. 变更（名称/分类等）：更新字段

**触发：** 每周日凌晨 03:00（通过 APScheduler）。

### 6.3 报告定时推送

**目标：** 将生成的报告自动推送到指定渠道。

**推送渠道：**
- 邮件（SMTP）
- 企业微信（Webhook）
- 飞书（Webhook）
- 钉钉（Webhook）

**配置：** `data_source_cfg` 表扩展或新增 `notification_config` 表。
